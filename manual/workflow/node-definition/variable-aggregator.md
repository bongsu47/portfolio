# 변수 집계기

변수 집계기 노드는 여러 경로에서 나온 변수를 하나로 모으는 노드입니다.

조건 분기나 의도 분류 노드로 여러 경로가 나뉜 후, 다시 하나의 흐름으로 합칠 때 사용합니다. 분기 후 합류 시 필수적인 노드입니다.

<figure><img src="../../../.gitbook/assets/image (648).png" alt="" width="306"><figcaption></figcaption></figure>

***

## 어떤 역할을 하나요?

조건 노드나 의도 분류 노드를 사용하면 워크플로우가 여러 경로로 나뉩니다. 이 중 실제로 실행되는 경로는 하나뿐이므로, 나머지 경로의 변수에는 값이 없습니다.

<figure><img src="../../../.gitbook/assets/image (9).png" alt="" width="375"><figcaption></figcaption></figure>

변수 집계기 노드는 여러 경로의 변수 중 실제로 값이 있는 것을 찾아 선택합니다.

예를 들어,

* 조건 노드에서 A 경로와 B 경로로 나뉜 후 → 실행된 쪽의 LLM 결과를 선택하여 답변 노드로 전달합니다
* 의도 분류로 3개 경로가 나뉜 후 → 실제 분류된 경로의 결과만 가져옵니다

분기 후 공통된 처리(예: 최종 답변 생성, 로그 기록)를 해야 할 때, 이 노드로 결과를 합류시킨 후 다음 노드로 연결합니다.

***

## 설정 방법

1. **분기된 2개 이상의 노드 다음에 변수 집계기 노드**를 추가한 뒤, **분기된 두 개의 노드**를 모두 해당 집계기 노드에 연결합니다.
2. 변수 집계기 노드를 클릭한 후, **입력 변수 설정**에서 **두 개의 LLM 노드 출력 변수를 모두 추가**합니다.
