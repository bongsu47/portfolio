# 코드

코드 노드는 직접 작성한 코드를 실행하여 데이터를 처리하는 노드입니다.

<figure><img src="../../../.gitbook/assets/image (658).png" alt="" width="312"><figcaption></figcaption></figure>

계산, 데이터 변환, 텍스트 가공 등 LLM이 아닌 정확한 로직이 필요할 때 사용합니다. LLM은 "대략적인" 처리에 강하지만, 정확한 계산이나 규칙 기반 처리에는 코드가 더 적합합니다.

***

## 어떤 역할을 하나요?

코드 노드는 프로그래밍 코드를 실행하여 원하는 결과를 만들어냅니다. LLM의 판단이 아닌, 정확한 규칙에 따라 데이터를 처리해야 할 때 유용합니다.

예를 들어,

* 숫자를 계산하거나 통계를 내야 할 때 → 코드로 정확하게 계산합니다
* 텍스트에서 특정 패턴을 추출해야 할 때 → 정규표현식으로 정확하게 추출합니다
* 데이터를 JSON, CSV 등 특정 형식으로 변환해야 할 때 → 코드로 변환 규칙을 작성합니다
* 날짜 계산, 문자열 가공 등 프로그래밍적 처리가 필요할 때

{% hint style="info" %}
#### 코드 노드 vs LLM 노드 vs 템플릿 노드

* **코드 노드**: 정확한 계산, 데이터 변환, 복잡한 로직. 프로그래밍 지식이 필요합니다.
* **LLM 노드**: 자연어 처리, 요약, 분류, 창의적 작업. 프롬프트만 작성하면 됩니다.
* **템플릿 노드**: 텍스트 조합, 포맷팅. Jinja2 문법으로 간단하게 처리할 수 있습니다.

"2 + 3은 5"처럼 정확한 결과가 필요하면 코드 노드를, "이 문서를 요약해줘"처럼 LLM의 판단이 필요하면 LLM 노드를 사용하세요.
{% endhint %}

***

## 설정 방법

<figure><img src="../../../.gitbook/assets/image (8).png" alt="" width="363"><figcaption></figcaption></figure>

### 1. 변수 매핑

코드에서 사용할 변수를 설정합니다. 이전 노드의 결과를 입력 변수로 연결하면, 코드 안에서 해당 변수 이름으로 바로 사용할 수 있습니다.

**추가** 버튼을 클릭하면 변수를 추가할 수 있습니다. 각 변수는 **변수 이름**과 **참조할 값**으로 구성됩니다.

사용 가능한 입력 타입:

| 타입                      | 설명                |
| ----------------------- | ----------------- |
| 문자열 (string)            | 텍스트 값             |
| 숫자 (number)             | 정수 또는 소수          |
| 객체 (object)             | JSON 형태의 구조화된 데이터 |
| 문자열 배열 (array\[string]) | 문자열 목록            |
| 숫자 배열 (array\[number])  | 숫자 목록             |
| 객체 배열 (array\[object])  | 구조화된 데이터 목록       |

***

### 2. 프로그래밍 언어 선택

코드 편집기 좌측 상단의 버튼을 클릭하여 **Python 3**와 **JavaScript** 중 원하는 언어를 선택할 수 있습니다. 언어를 변경하면 코드가 해당 언어의 기본 템플릿으로 초기화됩니다.

***

### 3. 코드 작성

코드 편집기에 처리할 로직을 작성합니다. 코드는 아래와 같은 형태로 작성해야 합니다.

* `main` 함수를 정의합니다
* 입력 변수를 매개변수로 받습니다
* 처리 결과를 `{}` 형태로 반환합니다
* 반환하는 키 이름이 아래에서 정의할 **출력 변수 이름과 동일**해야 합니다

**예시) 두 숫자를 더해서 결과를 반환**:

```python
# Python
def main(price: int, quantity: int) -> dict:
    return {
        "total": price * quantity,
    }
```

```javascript
// JavaScript
function main({ price, quantity }) {
    return {
        total: price * quantity
    }
}
```

{% hint style="info" %}
#### 코드 작성이 어렵다면?

미소 AI(자동 생성)를 활용하면, 원하는 처리를 자연어로 설명하는 것만으로 미소에서 지원하는 형식에 맞는 코드를 쉽게 작성할 수 있습니다. 예를 들어 "두 날짜 사이의 일수를 계산해줘"라고 입력하면 적절한 코드를 자동으로 생성합니다.
{% endhint %}

코드는 안전한 샌드박스 환경에서 실행됩니다.

***

### 4. 출력 변수

코드가 반환할 결과의 이름과 타입을 정의합니다. **추가** 버튼을 클릭하여 출력 변수를 추가합니다.

| 출력 타입                   | 설명               |
| ----------------------- | ---------------- |
| 문자열 (string)            | 문자열 결과           |
| 숫자 (number)             | 정수 또는 소수         |
| 객체 (object)             | JSON 형태의 구조화된 결과 |
| 문자열 배열 (array\[string]) | 문자열 목록           |
| 숫자 배열 (array\[number])  | 숫자 목록            |
| 객체 배열 (array\[object])  | 구조화된 데이터 목록      |

반환된 값이 정의한 타입과 일치하지 않으면 오류가 발생합니다. 예를 들어, 출력 타입을 숫자로 정의했는데 문자열을 반환하면 실패합니다.

***

## 실행 제한

코드는 안전한 샌드박스 환경에서 실행되며, 아래 제한이 적용됩니다.

| 항목         | 제한값       |
| ---------- | --------- |
| 실행 시간      | 최대 60초    |
| 문자열 배열 크기  | 최대 100개   |
| 숫자 배열 크기   | 최대 1,000개 |
| 객체 배열 크기   | 최대 100개   |
| 객체 중첩 깊이   | 최대 5단계    |
| 숫자 소수점 자릿수 | 최대 20자리   |

제한을 초과하면 오류가 발생하므로, 필요한 데이터만 반환하도록 작성하세요.

## 활용 예시

#### 1. 문자열을 배열로 변환하여 반복 노드에 전달

**상황**: 사용자가 입력한 쉼표로 구분된 텍스트를 배열로 변환하여 반복 노드에서 항목별로 처리하고 싶을 때

* **입력 변수**: `tags`(문자열) - 예: `"마케팅, 디자인, 개발"`
* **출력 변수**: `tag_list`(문자열 배열)

```python
def main(tags: str) -> dict:
    tag_list = [t.strip() for t in tags.split(",") if t.strip()]
    return {
        "tag_list": tag_list,
    }
```

**결과**: `["마케팅", "디자인", "개발"]` → 이후 반복 노드에서 각 항목을 순회하며 처리

***

#### 2. 외부 API 호출

**상황**: 외부 서비스의 API를 호출하여 데이터를 가져오고 싶을 때

* **입력 변수**: `keyword`(문자열)
* **출력 변수**: `result`(객체)
* **의존성 패키지**: `requests`

```python
import requests

def main(keyword: str) -> dict:
    response = requests.get(
        "https://api.example.com/search",
        params={"q": keyword},
        timeout=10,
    )
    return {
        "result": response.json(),
    }
```

***

#### 3. LLM의 JSON 출력에서 변수 분리

**상황**: LLM 노드에서 JSON 형태로 생성한 결과를 파싱하여 각 값을 별도 변수로 분리하고 싶을 때

* **입력 변수**: `llm_output`(문자열) - 예: `'{"summary": "...", "category": "기술", "keywords": ["AI", "자동화"]}'`
* **출력 변수**: `summary`(문자열), `category`(문자열), `keywords`(문자열 배열)

```python
import json

def main(llm_output: str) -> dict:
    parsed = json.loads(llm_output)
    return {
        "summary": parsed["summary"],
        "category": parsed["category"],
        "keywords": parsed["keywords"],
    }
```

**결과**: LLM이 한 번에 생성한 JSON을 개별 변수로 분리하여, 이후 노드에서 각각 독립적으로 활용
