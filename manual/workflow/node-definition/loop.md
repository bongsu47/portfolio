# 루프

루프 노드는 **특정 조건이 충족될 때까지** 작업을 반복 실행하는 노드입니다.\
순서대로 처리할 수도 있고, 원하는 결과가 나올 때까지 계속 실행할 수 있습니다.

<figure><img src="../../../.gitbook/assets/image (652).png" alt=""><figcaption></figcaption></figure>

***

## 반복 노드와 무엇이 다른가요?

반복 노드와 루프 노드는 모두 "작업을 여러 번 실행"하지만, 동작 방식이 근본적으로 다릅니다.

**반복 노드**는 배열(목록)을 입력으로 받아, 각 항목에 대해 같은 작업을 수행합니다.\
배열이 5개면 5번, 10개면 10번. 횟수가 미리 정해져 있습니다.

**루프 노드**는 배열 없이도 동작합니다.\
"성공할 때까지", "기준을 충족할 때까지"처럼 **조건이 만족될 때까지** 같은 작업을 반복합니다. 몇 번 반복해야 할지 미리 알 수 없는 상황에 적합합니다.

|        | 반복 노드              | 루프 노드                |
| ------ | ------------------ | -------------------- |
| 입력     | 배열 (목록) 필수         | 배열 없이도 사용 가능         |
| 반복 횟수  | 배열 크기만큼 (정해져 있음)   | 조건 충족 시까지 (유동적)      |
| 종료 조건  | 배열의 모든 항목 처리 완료    | 중단 조건 충족 또는 최대 횟수 도달 |
| 병렬 실행  | 지원                 | 미지원                  |
| 적합한 상황 | "이 목록의 각 항목을 처리해줘" | "성공할 때까지 재시도해줘"      |

***

## 어떤 역할을 하나요?

루프 노드는 설정한 조건이 만족될 때까지, 또는 최대 횟수에 도달할 때까지 작업을 반복합니다.

예를 들어,

* AI의 답변 품질이 기준을 충족할 때까지 다시 생성
* 사용자 질문을 충족하는 데이터를 모두 수집할 때까지 반복 수집
* 지식 인덱싱이 완료될 때까지 상태를 주기적으로 확인

***

## 설정 방법

### 1. 최대 반복 횟수

<figure><img src="../../../.gitbook/assets/image (653).png" alt=""><figcaption></figcaption></figure>

루프가 실행될 수 있는 최대 횟수를 설정합니다.\
중단 조건이 충족되지 않더라도, 이 횟수에 도달하면 루프가 자동으로 종료됩니다.

* 기본값: 10회
* 설정 범위: 1\~100회
* 슬라이더 또는 숫자 입력으로 설정

{% hint style="info" %}
#### 안전장치 역할

최대 반복 횟수는 무한 루프를 방지하는 안전장치입니다. 중단 조건에 논리적 오류가 있어서 조건이 영원히 충족되지 않더라도, 최대 횟수에 도달하면 자동으로 멈춥니다.

또한 루프의 **최대 실행 시간은 5분**입니다. 횟수와 관계없이 5분을 초과하면 자동으로 중단됩니다.
{% endhint %}

***

### 2. 루프 변수

<figure><img src="../../../.gitbook/assets/image (654).png" alt=""><figcaption></figcaption></figure>

루프 안에서 계속 변하는 값을 기억해두는 ‘메모장’ 같은 변수입니다.\
처음 루프가 시작될 때 기본값을 적어두고, 반복이 돌 때마다 필요한 값으로 덮어쓰면서 상태를 이어갑니다.

**+ 추가** 버튼을 클릭하면 변수를 추가할 수 있습니다.

<figure><img src="../../../.gitbook/assets/image (656).png" alt=""><figcaption></figcaption></figure>

루프 이후 노드에서 루프 내부에서 처리된 데이터를 활용하려면, 해당 값을 루프 변수에 저장하기 위해 변수 할당 노드를 사용해야 합니다.

루프 변수 외에도 `index` 변수가 자동으로 제공됩니다. 현재 몇 번째 반복인지를 나타내며, 0부터 시작합니다.

{% hint style="info" %}
#### 루프 변수를 활용하는 일반적인 패턴

**1. 성공 여부 추적**

* `success`(불리언)를 `false`로 시작
* API 호출 성공 시 변수 할당으로 `true`로 변경
* 중단 조건: `success` = `true`

**2. 결과 수집**

* `results`(문자열 배열)를 빈 배열 `[]`로 시작
* 매 반복마다 결과를 배열에 추가
* 중단 조건: 원하는 수만큼 수집되면 종료

**3. 카운터**

* `count`(숫자)를 `0`으로 시작
* 특정 조건 만족 시 1씩 증가
* 중단 조건: `count` ≥ 목표값
{% endhint %}

***

### 3. 중단 조건

루프를 멈출 조건을 설정합니다. 각 반복이 끝날 때마다 조건을 검사하여, 충족되면 즉시 루프가 종료됩니다.

**+ 추가** 버튼을 클릭하면 조건을 추가할 수 있습니다.

각 조건은 **비교할 변수**, **비교 연산자**, **비교 값**으로 구성됩니다. 비교 연산자는 조건 노드와 동일합니다.

여러 조건을 설정한 경우, **AND/OR** 버튼으로 조건 간의 관계를 선택할 수 있습니다.

| 조건 연산자  | 설명              | 예시                                                |
| ------- | --------------- | ------------------------------------------------- |
| **AND** | 모든 조건이 충족되어야 중단 | `success = true` AND `count ≥ 3` → 둘 다 만족해야 루프 종료 |
| **OR**  | 하나라도 충족되면 중단    | `success = true` OR `count ≥ 5` → 하나만 만족해도 루프 종료  |

{% hint style="info" %}
#### 중단 조건이 없으면?

중단 조건을 설정하지 않으면, 최대 반복 횟수까지 무조건 실행됩니다.\
이 경우 "정확히 N번 반복"하는 용도로 사용할 수 있습니다.
{% endhint %}

***

### 4. 루프 종료 노드

위의 **중단 조건**과 동일한 역할을 하지만, 설정이 아닌 **노드 형태**로 루프 내부에 직접 배치할 수 있습니다.\
이 노드가 실행되면 루프가 **즉시** 종료됩니다. 추가 설정은 없습니다.

<figure><img src="../../../.gitbook/assets/image (657).png" alt=""><figcaption></figcaption></figure>

중단 조건은 매 반복이 끝난 뒤 자동으로 검사되는 반면, 루프 종료 노드는 **루프 흐름의 원하는 위치에 자유롭게 배치**할 수 있습니다. 조건 노드와 조합하면 "특정 상황에서만 루프를 탈출"하는 종료 로직을 노드 흐름으로 구성할 수 있어, 워크플로우의 흐름을 시각적으로 파악하기 쉽습니다.

중단 조건과 함께 사용할 수도 있습니다. 루프 종료 노드가 먼저 실행되면 중단 조건 검사 없이 즉시 종료됩니다.

***

## 출력 변수

루프가 종료되면 아래 값들이 출력됩니다.

| 변수           | 설명                                   |
| ------------ | ------------------------------------ |
| 루프 변수들       | 정의한 각 루프 변수의 **최종 값** (마지막 반복 시점의 값) |
| `loop_round` | 실제로 실행된 반복 횟수                        |

예를 들어, 최대 반복 10회로 설정했지만 3번째 반복에서 중단 조건이 충족되면:

* `loop_round` = 3
* 루프 변수들은 3번째 반복까지 업데이트된 값
