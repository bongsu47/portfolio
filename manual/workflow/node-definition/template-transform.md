# 템플릿

템플릿 노드는 여러 변수를 조합하여 하나의 텍스트를 만드는 노드입니다.

<figure><img src="../../../.gitbook/assets/image (13).png" alt="" width="297"><figcaption></figcaption></figure>

쉽게 말해 **빈칸 채우기**와 같습니다. 미리 작성해 둔 틀에 변수 값을 끼워 넣어 원하는 형식의 텍스트를 완성합니다. 이메일 제목을 만들거나, 지식 검색 결과를 보기 좋게 정리하거나, LLM에 전달할 프롬프트를 동적으로 구성할 때 유용합니다.

{% hint style="info" %}
#### 템플릿 vs 코드 vs LLM, 어떤 노드를 써야 하나요?

* **템플릿 노드**: 변수를 조합하여 텍스트를 만들 때. 가장 가볍고 빠릅니다.
* **코드 노드**: 계산, 데이터 변환 등 로직이 필요할 때. 프로그래밍 언어를 사용합니다.
* **LLM 노드**: LLM이 내용을 이해하고 새로운 텍스트를 생성해야 할 때. 비용과 시간이 가장 많이 듭니다.

단순히 변수를 합치거나 형식을 정하는 작업이라면, LLM을 쓰는 것보다 템플릿 노드가 훨씬 효율적입니다.
{% endhint %}

***

## 어떤 역할을 하나요?

템플릿 노드는 미리 작성한 틀에 변수 값을 끼워 넣어 원하는 형식의 텍스트를 완성합니다.

예를 들어,

* 지식 검색 결과를 보기 좋게 정리하여 LLM에 전달할 때
* 사용자 이름, 문의 내용 등을 조합해 일관된 응대 메시지를 만들 때
* 조건에 따라 다른 내용을 포함한 프롬프트를 동적으로 구성할 때
* 검색 결과를 HTML 표나 목록 형식으로 구조화할 때

LLM 호출 없이 동작하므로 비용이 없고, 결과가 항상 일관됩니다.

***

## 설정 방법

<figure><img src="../../../.gitbook/assets/image (14).png" alt="" width="368"><figcaption></figcaption></figure>

### 1. 입력 변수 설정

템플릿에서 사용할 변수를 추가합니다. **추가** 버튼을 클릭하여 변수 이름을 지정하고, 이전 노드의 값을 연결합니다.

예를 들어, 사용자 입력을 `query`라는 이름으로, 지식 검색 결과를 `results`라는 이름으로 추가할 수 있습니다.

***

### 2. 템플릿 작성

틀이 되는 텍스트를 작성하고, 변수를 넣을 위치에 `{{ 변수이름 }}`을 입력합니다.

```
안녕하세요, {{ name }}님!
오늘 문의하신 내용은 "{{ query }}" 입니다.
```

위 템플릿에서 `name`이 "홍길동", `query`가 "환불 방법"이라면, 결과는 다음과 같습니다.

```
안녕하세요, 홍길동님!
오늘 문의하신 내용은 "환불 방법" 입니다.
```

{% hint style="info" %}
#### 변수 빠르게 삽입하기

코드 편집기에서 `{{` 나 `/` 를 입력하면 사용 가능한 변수 목록이 자동으로 나타납니다. 목록에서 원하는 변수를 선택하면 자동으로 삽입됩니다.
{% endhint %}

***

## 자주 쓰는 문법

### 조건에 따라 다른 내용 표시

`{% if %}` \~ `{% endif %}`로 조건에 따라 다른 내용을 넣을 수 있습니다.

```
{% if results | length > 0 %}
검색 결과가 {{ results | length }}건 있습니다.
{% else %}
검색 결과가 없습니다. 다른 키워드로 시도해보세요.
{% endif %}
```

한 줄로 간단하게 쓸 수도 있습니다.

```
상태: {{ "VIP 고객" if is_vip else "일반 고객" }}
```

***

### 목록 반복하여 나열

`{% for %}` \~ `{% endfor %}`로 배열의 각 항목을 순서대로 출력할 수 있습니다.

```
{% for item in results %}
{{ loop.index }}. {{ item.title }}
{% endfor %}
```

결과:

```
1. 환불 정책 안내
2. 반품 절차 가이드
3. 교환 방법
```

배열이 비어있을 때 대체 텍스트를 표시할 수도 있습니다.

```
{% for item in results %}
- {{ item.title }}
{% else %}
검색 결과가 없습니다.
{% endfor %}
```

***

### 필터로 값 변환

`|` (파이프) 기호 뒤에 필터를 붙이면 값을 변환할 수 있습니다.

| 필터                | 설명           | 예시                                | 결과                 |
| ----------------- | ------------ | --------------------------------- | ------------------ |
| `default`         | 빈 값일 때 기본값   | `{{ name \| default("미입력") }}`    | `미입력`              |
| `length`          | 항목 수         | `{{ items \| length }}`           | `3`                |
| `join`            | 배열을 하나로 합치기  | `{{ items \| join(", ") }}`       | `사과, 바나나, 오렌지`     |
| `truncate`        | 길이 제한        | `{{ text \| truncate(50) }}`      | `긴 텍스트를 50자까지만...` |
| `upper` / `lower` | 대/소문자 변환     | `{{ "hello" \| upper }}`          | `HELLO`            |
| `replace`         | 텍스트 치환       | `{{ text \| replace("A", "B") }}` | A를 B로 치환           |
| `round`           | 반올림          | `{{ 3.1415 \| round(2) }}`        | `3.14`             |
| `tojson`          | JSON 문자열로 변환 | `{{ data \| tojson }}`            | `{"key": "value"}` |

필터를 여러 개 연결할 수도 있습니다.

```
{{ results | map(attribute="title") | join(", ") }}
```

결과: `환불 정책 안내, 반품 절차 가이드, 교환 방법`

***

### 객체에서 값 꺼내기

변수가 여러 속성을 가진 객체일 때, 점(`.`)으로 원하는 값을 꺼낼 수 있습니다.

```
이름: {{ user.name }}
이메일: {{ user.email }}
```

## 활용 예시

#### 1. 지식 검색 결과를 LLM에 전달하기

지식 검색 노드의 결과를 보기 좋게 정리하여 LLM에 전달하면, 더 정확한 답변을 얻을 수 있습니다.

```
아래 자료를 참고하여 사용자의 질문에 답변해주세요.

사용자 질문: {{ query }}

{% for doc in results %}
---
[참고자료 {{ loop.index }}] {{ doc.title }}
{{ doc.content }}
{% endfor %}
```

***

#### 2. 조건에 따라 다른 프롬프트 구성하기

사용자의 상황에 따라 LLM에 전달할 지시문을 다르게 구성할 수 있습니다.

```
{% if category == "환불" %}
고객이 환불을 요청했습니다. 환불 정책에 맞게 안내해주세요.
{% elif category == "배송" %}
고객이 배송에 대해 문의했습니다. 배송 현황을 안내해주세요.
{% else %}
고객 문의에 친절하게 응대해주세요.
{% endif %}

고객 메시지: {{ message }}
```

***

#### 3. 고객 응대 메시지 만들기

변수를 조합하여 일관된 형식의 응대 메시지를 만들 수 있습니다.

```
{{ name | default("고객") }}님, 안녕하세요.

문의하신 "{{ query }}" 건에 대해 안내드립니다.

{{ answer }}

추가 궁금한 점이 있으시면 언제든 문의해주세요.
감사합니다.
```

***

#### 4. 여러 검색 결과를 번호 매긴 목록으로 정리하기

```
총 {{ results | length }}건의 결과를 찾았습니다.

{% for item in results %}
{{ loop.index }}. {{ item.title }}
   - {{ item.content | truncate(100) }}
{% else %}
관련 자료를 찾지 못했습니다.
{% endfor %}
```

***

#### 5. HTML 형식의 응답 만들기

템플릿을 활용하면 HTML 형식의 깔끔한 응답을 만들 수 있습니다. 표, 목록, 링크 등을 포함한 구조화된 응답이 필요할 때 유용합니다.

```
<h2>{{ query }}에 대한 검색 결과</h2>

<table>
  <tr>
    <th>번호</th>
    <th>제목</th>
    <th>내용</th>
  </tr>
  {% for item in results %}
  <tr>
    <td>{{ loop.index }}</td>
    <td>{{ item.title }}</td>
    <td>{{ item.content | truncate(80) }}</td>
  </tr>
  {% endfor %}
</table>

{% if results | length == 0 %}
<p>검색 결과가 없습니다.</p>
{% endif %}
```

링크나 강조 표시를 넣을 수도 있습니다.

```
<p>안녕하세요, <strong>{{ name | default("고객") }}</strong>님.</p>

<ul>
  {% for item in items %}
  <li>{{ item.title }} — {{ item.description }}</li>
  {% endfor %}
</ul>

<p>자세한 내용은 <a href="{{ link }}">여기</a>를 참고해주세요.</p>
```

***

#### 6. 빈 값에 기본값 설정하기

변수가 비어있을 때를 대비하여 기본값을 설정하면, 빈 칸 없이 깔끔한 결과를 만들 수 있습니다.

```
이름: {{ name | default("미입력") }}
부서: {{ department | default("소속 없음") }}
연락처: {{ phone | default("등록된 연락처 없음") }}
```

***

### 출력 변수

| 변수       | 설명              |
| -------- | --------------- |
| `output` | 템플릿이 완성된 최종 텍스트 |

{% hint style="info" %}
#### 알아두면 좋은 점

* 출력 텍스트의 최대 길이는 **80,000자**입니다. 이를 초과하면 오류가 발생합니다.
* 변수가 존재하지 않을 때 오류를 방지하려면 `| default("")` 필터를 활용하세요.
* 단순히 변수 하나를 그대로 전달하는 용도라면, 템플릿 노드 대신 직접 변수를 참조하는 것이 더 간단합니다.
* 계산이나 복잡한 로직이 필요하다면, 코드 노드를 사용하는 것이 더 적합합니다.
{% endhint %}
