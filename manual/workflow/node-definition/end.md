# 끝/답변

끝(답변) 노드는 워크플로우의 마침점이거나, 처리 결과를 사용자에게 전달하는 노드입니다.

워크플로우가 모든 처리를 마치고 최종 결과를 내보내는 데 사용되며, 설정에 따라 중간 단계에서 사용자에게 메시지를 전달하는 용도로도 활용할 수 있습니다.

이 노드는 워크플로우 설정에 따라 이름이 달라집니다.

<figure><img src="../../../.gitbook/assets/image (612).png" alt="" width="289"><figcaption></figcaption></figure>

| 워크플로우 설정        | 노드 이름  | 역할               |
| --------------- | ------ | ---------------- |
| 일반 워크플로우        | **끝**  | 최종 출력 변수를 정의합니다  |
| 채팅 설정이 켜진 워크플로우 | **답변** | 사용자에게 메시지를 전달합니다 |

기능은 다르지만, 둘 다 **워크플로우의 결과를 내보내는 노드**라는 점은 같습니다.

***

## 어떤 역할을 하나요?

### 1 .끝 (일반 워크플로우)

끝 노드는 워크플로우의 실행을 종료하고, 최종 출력값을 정의하는 역할입니다. 이전 노드들에서 만들어진 결과 중 어떤 값을 최종 결과로 내보낼지 선택합니다.

예를 들어,

* LLM 노드에서 생성한 답변을 최종 결과로 내보낼 수 있습니다
* 코드 노드에서 계산한 값을 최종 결과로 내보낼 수 있습니다

또한 조건 분기 등이 있는 경우, 워크플로우는 분기된 경로 중 한 방향으로만 실행되므로, 각 분기 흐름마다 별도의 끝 노드를 추가하여 서로 다른 결과를 정의할 수 있습니다.

<figure><img src="../../../.gitbook/assets/image (615).png" alt="" width="563"><figcaption></figcaption></figure>

워크플로우는 끝 노드에 도달하면 실행이 완료됩니다.

### 2. 답변 (채팅 워크플로우)

답변 노드는 워크플로우의 처리 결과를 사용자에게 메시지로 전달하는 역할입니다.\
이전 노드들에서 만들어진 결과를 조합하여 하나의 답변으로 보여줍니다.

또한 답변 노드는 워크플로우의 끝에서만 사용하는 것이 아니라, 중간에도 여러 개를 배치할 수 있습니다.\
이를 통해 워크플로우 진행 중간중간에 특정 메시지를 사용자에게 전달할 수 있으며, 상태 안내나 중간 결과를 보여주는 용도로 활용할 수 있습니다.

<figure><img src="../../../.gitbook/assets/image (613).png" alt="" width="563"><figcaption></figcaption></figure>

예를 들어,

* LLM이 생성한 답변을 사용자에게 보여줄 수 있습니다
* 지식 검색 결과와 LLM 답변을 함께 조합하여 전달할 수 있습니다
* 파일을 함께 첨부하여 전달할 수도 있습니다
* 처리 진행 상황이나 중간 결과를 사용자에게 안내할 수 있습니다

답변은 실시간으로 스트리밍되어, 사용자가 기다리지 않고 바로 읽을 수 있습니다.

***

## 설정 방법

### 1. 끝 (일반 워크플로우)

일반 워크플로우에서 사용하는 끝 노드는 아래 단계에 따라 설정할 수 있습니다.

#### 1) 출력 변수 추가

* 최종 결과로 내보낼 변수를 설정합니다. 필요에 따라 하나 이상의 변수를 추가할 수 있습니다.
* 각 변수에는 이름을 지정할 수 있으며, 이 이름이 워크플로우의 최종 출력 키가 됩니다.

{% hint style="info" %}
### 출력 변수를 꼭 추가해야하나요?

출력 변수는 반드시 설정해야 하는 것은 아닙니다.

예를 들어, 메일 발송과 같은 작업을 수행하는 경우에는 처리 성공 여부만 간단히 전달하거나, 별도의 출력 없이 실행만 수행할 수도 있습니다.

다만, 외부 API와 연동하거나 에이전트가 워크플로우를 도구로 호출하는 경우에는 끝 노드에 설정된 출력 변수만 최종 결과로 전달됩니다.이러한 경우에는 필요한 값을 출력 변수로 지정해주는 것이 중요합니다.
{% endhint %}

#### 2) 변수 선택

* 이전 노드에서 생성된 변수 중 원하는 값을 선택합니다. 워크플로우의 마지막 단계에서 어떤 값을 결과로 사용할지 결정하는 과정입니다.
* 여러 변수를 동시에 선택하여 내보낼 수 있습니다.

### 2. 답변 (채팅 워크플로우)

채팅 워크플로우에서 사용하는 답변 노드는 아래와 같이 설정할 수 있습니다.

* 직접 메시지를 입력하거나 `/`를 사용하여 변수를 삽입할 수 있습니다.
* 이전 노드에서 생성된 변수를 선택하여 사용 가능합니다.
* 메시지와 변수를 함께 사용하거나, 각각 독립적으로 활용할 수 있습니다.
