# 지식 추가

지식 추가 노드는 워크플로우 실행 중에 **지식에 새로운 내용을 추가하는** 노드입니다.\
대화 과정에서 생성된 텍스트나 사용자가 업로드한 파일을 지식으로 저장하여, 이후 검색에 활용할 수 있습니다.

<figure><img src="../../../.gitbook/assets/image (40).png" alt="" width="290"><figcaption></figcaption></figure>

***

## 어떤 역할을 하나요?

지식 추가 노드는 **워크플로우가 실행되는 도중에 지식을 동적으로 추가**합니다.\
미리 등록해 둔 지식만 사용하는 것이 아니라, 대화가 진행되면서 새로운 정보를 쌓아갈 수 있습니다.

예를 들어,

* 상담 내용을 요약한 결과를 지식에 저장하여, 다음 상담 시 참고 자료로 활용
* 사용자가 업로드한 문서를 자동으로 지식에 등록하여, 이후 검색 가능하도록 구성
* 매일 수집되는 데이터를 하나의 지식에 계속 누적

추가된 내용은 인덱싱이 완료되면 지식 검색 노드에서 바로 검색할 수 있습니다.

***

## 언제 사용하나요?

지식 추가 노드는 **워크플로우 안에서 지식을 자동으로 쌓아야 할 때** 사용합니다.

* 대화 결과나 AI가 생성한 내용을 지식으로 저장하고 싶을 때
* 사용자가 업로드한 파일을 지식에 자동 등록하고 싶을 때
* 반복 실행되는 워크플로우에서 결과를 하나의 지식에 계속 누적하고 싶을 때

{% hint style="info" %}
#### 지식 추가와 지식 검색의 차이

지식 검색은 이미 등록된 지식에서 정보를 **찾아오는** 노드입니다.\
지식 추가는 지식에 새로운 정보를 **넣는** 노드입니다.\
두 노드를 함께 사용하면 "정보를 쌓으면서 동시에 활용하는" 워크플로우를 만들 수 있습니다.
{% endhint %}

***

## 설정 방법

지식 추가 노드를 클릭하면 설정 패널이 열립니다.\
패널은 **입력 변수**와 **도구 설정** 두 영역으로 나뉩니다.

***

<figure><img src="../../../.gitbook/assets/image (41).png" alt="" width="325"><figcaption></figcaption></figure>

### 입력 변수

입력 변수는 워크플로우에서 흘러온 값을 받는 영역입니다.\
각 필드에 직접 텍스트를 입력하거나, `/`를 입력하여 이전 노드의 변수를 선택할 수 있습니다.

#### **1. 지식 이름**

추가할 대상 지식의 이름입니다.\
도구 설정에서 기존 지식을 선택하지 않은 경우, 여기에 입력한 이름으로 **새 지식이 자동 생성**됩니다.

변수를 연결하면 워크플로우 실행마다 다른 이름으로 지식을 생성할 수도 있습니다.

#### **2. 지식 설명**

새로 생성되는 지식에 대한 설명입니다.\
기존 지식을 선택한 경우에는 사용되지 않습니다.

#### **3. 텍스트 콘텐츠**

지식에 추가할 텍스트 내용입니다.\
이전 노드에서 생성된 텍스트(예: LLM의 요약 결과, 코드 노드의 출력)를 변수로 연결하는 것이 일반적입니다.

직접 텍스트를 입력할 수도 있고, 변수와 텍스트를 조합할 수도 있습니다.

#### **4. 콘텐츠 파일명**

텍스트 콘텐츠가 저장될 때 사용할 파일명입니다.\
입력하지 않으면 기본값인 `content.txt`로 저장됩니다.\
`.txt` 확장자는 생략해도 자동으로 붙습니다. 예를 들어 `상담기록`이라고 입력하면 `상담기록.txt`로 저장됩니다.

**기존 문서에 추가** 옵션을 사용할 때 파일명이 중요해집니다.\
기존 지식에서 같은 이름의 문서를 찾아 해당 문서에 내용을 이어붙이기 때문입니다.\
이 옵션을 사용하지 않는다면 파일명은 단순 식별용이므로 크게 신경 쓰지 않아도 됩니다.

#### **5. 기존 문서에 추가**

토글을 켜면, 같은 파일명의 문서가 이미 지식에 있을 때 **새 문서를 만들지 않고 기존 문서에 내용을 이어붙입니다.**

|              | 토글 꺼짐 (기본)        | 토글 켜짐                       |
| ------------ | ----------------- | --------------------------- |
| 동작           | 항상 새 문서로 추가       | 같은 파일명의 문서가 있으면 해당 문서에 이어붙임 |
| 같은 파일명이 있을 때 | 동일한 이름의 문서가 2개 생김 | 기존 문서에 내용 추가                |
| 같은 파일명이 없을 때 | 새 문서 생성           | 새 문서 생성 (토글 꺼짐과 동일)         |

반복적으로 실행되는 워크플로우에서 결과를 하나의 문서에 계속 누적하고 싶을 때 유용합니다.

***

### 도구 설정

도구 설정은 이 노드의 기본 동작을 지정하는 영역입니다.

#### **1. 지식 선택**

내용을 추가할 **기존 지식**을 드롭다운에서 선택합니다.\
현재 워크스페이스에 등록된 지식 목록이 나타납니다.

* 기존 지식을 선택하면 → 해당 지식에 문서가 추가됩니다
* 선택하지 않으면 → 입력 변수의 "지식 이름"으로 새 지식이 생성됩니다

#### **2. 지식 이름**

도구 설정에서 기존 지식을 선택하지 않았을 때 나타나는 필드입니다.\
새로 생성할 지식의 이름을 지정합니다. `/`를 입력하여 변수를 선택할 수 있습니다.

#### **3. 파일**

지식에 추가할 **파일이 담긴 변수**를 선택합니다.\
시작 노드에서 파일 입력을 받았거나, 이전 노드에서 파일을 처리한 결과가 있을 때 해당 변수를 연결합니다.

텍스트 콘텐츠와 파일을 동시에 사용할 수도 있습니다. 둘 다 입력하면 텍스트와 파일이 모두 지식에 추가됩니다.

***

#### 기존 지식에 추가할까요, 새 지식을 만들까요?

|        | 기존 지식에 추가                | 새 지식 생성                       |
| ------ | ------------------------ | ----------------------------- |
| 설정 방법  | 도구 설정 > 지식 선택에서 기존 지식 선택 | 지식 선택을 비워두고, 입력 변수 > 지식 이름 입력 |
| 적합한 경우 | 고정된 하나의 지식에 계속 내용을 쌓을 때  | 실행할 때마다 새로운 지식을 만들어야 할 때      |
| 예시     | 상담 기록을 하나의 지식에 누적        | 고객별로 별도의 지식 생성                |

***

#### 텍스트 콘텐츠와 파일, 어떤 걸 사용하나요?

|        | 텍스트 콘텐츠                   | 파일                            |
| ------ | ------------------------- | ----------------------------- |
| 입력 위치  | 입력 변수 영역                  | 도구 설정 영역                      |
| 사용 상황  | LLM이 생성한 텍스트, 코드 노드의 결과 등 | 사용자가 업로드한 파일, 외부에서 받아온 파일     |
| 저장 형식  | .txt 파일로 변환되어 저장          | 원본 파일 형식 그대로 저장 (PDF, DOCX 등) |
| 적합한 경우 | 워크플로우 내에서 만들어진 텍스트를 저장할 때 | 파일 자체를 지식에 등록할 때              |

둘 중 하나만 입력해도 되고, 둘 다 입력해도 됩니다.\
단, 텍스트 콘텐츠와 파일 중 **아무것도 입력하지 않으면** 노드가 실행을 건너뜁니다.

***

### 출력 변수

| 변수             | 타입     | 설명            |
| -------------- | ------ | ------------- |
| `status`       | String | 처리 상태         |
| `dataset_id`   | String | 지식의 고유 ID     |
| `dataset_name` | String | 지식 이름         |
| `document_ids` | Array  | 추가된 문서의 ID 목록 |
| `message`      | String | 처리 결과 메시지     |

`status`는 다음 중 하나의 값을 가집니다.

| 상태           | 의미                                        |
| ------------ | ----------------------------------------- |
| `processing` | 새 문서가 추가되었고, 인덱싱이 진행 중입니다                 |
| `completed`  | 기존 문서에 내용이 추가 완료되었습니다 (기존 문서에 추가 옵션 사용 시) |
| `skipped`    | 텍스트 콘텐츠와 파일이 모두 비어있어 실행을 건너뛰었습니다          |

{% hint style="warning" %}
#### 인덱싱이 완료되어야 검색할 수 있습니다

새 문서를 추가한 경우(`processing`), 인덱싱이 완료될 때까지 해당 내용은 지식 검색에 나타나지 않습니다.\
인덱싱에 걸리는 시간은 문서의 크기에 따라 다릅니다. 인덱싱 상태를 확인하려면 **인덱싱 상태 확인** 노드를 사용하세요.

기존 문서에 추가한 경우(`completed`)에는 즉시 검색이 가능합니다.
{% endhint %}
