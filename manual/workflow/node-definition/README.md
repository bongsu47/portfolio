# 노드 알아보기

노드는 MISO의 모듈 및 챗모듈에서 **동작을 정의하는 최소 단위**입니다. 다양한 노드를 조합하여 원하는 애플리케이션을 구성할 수 있습니다.

각 노드는 하나의 역할을 담당하며, 노드와 노드를 연결하면 데이터가 순서대로 흘러가는 워크플로우가 만들어집니다.

예를 들어, 사용자의 질문을 받아 → 지식을 검색하고 → AI가 답변을 생성하여 → 사용자에게 전달하는 흐름을 노드 4개로 구성할 수 있습니다.

***

#### 기본 노드

워크플로우의 시작과 끝을 담당하는 노드입니다.

| 노드             | 설명                                   |
| -------------- | ------------------------------------ |
| [시작](start.md) | 워크플로우의 출발점. 사용자 입력을 받아 다음 노드로 전달합니다. |
| [끝/답변](end.md) | 워크플로우의 마침점. 처리 결과를 사용자에게 전달합니다.      |

#### AI 노드

AI 모델을 활용하여 텍스트를 생성하거나 분석하는 노드입니다.

| 노드                              | 설명                                    |
| ------------------------------- | ------------------------------------- |
| [LLM](llm.md)                   | AI 모델에게 프롬프트를 전달하여 텍스트를 생성합니다.        |
| [의도 분류](question-classifier.md) | 사용자의 입력이 어떤 의도인지 AI가 판별합니다.           |
| [변수 추출](parameter-extractor.md) | 텍스트에서 원하는 정보를 AI가 추출하여 각각의 변수로 분리합니다. |

#### 지식 노드

등록된 지식 데이터를 검색하고 관리하는 노드입니다.

| 노드                                 | 설명                             |
| ---------------------------------- | ------------------------------ |
| [지식 검색](knowledge-retrieval.md)    | 등록된 지식에서 질문과 관련된 내용을 검색합니다.    |
| [지식 추가](knowledge-add.md)          | 워크플로우 실행 중 새로운 지식을 동적으로 추가합니다. |
