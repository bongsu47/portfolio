# 인덱싱 상태 확인

인덱싱 상태 확인 노드는 지식에 추가한 문서의 **인덱싱 진행 상태를 확인하는** 노드입니다.\
지식 추가 노드로 문서를 추가한 뒤, 인덱싱이 완료되었는지 워크플로우 안에서 확인할 수 있습니다.

<figure><img src="../../../.gitbook/assets/image (43).png" alt="" width="290"><figcaption></figcaption></figure>

***

## 어떤 역할을 하나요?

지식에 문서를 추가하면 바로 검색할 수 있는 것이 아닙니다.\
문서를 적절한 크기로 나누고, 검색 가능한 형태로 변환하는 **인덱싱** 과정이 필요합니다.\
이 과정은 문서의 크기에 따라 수 초에서 수십 분까지 걸릴 수 있습니다.

인덱싱 상태 확인 노드는 이 과정이 **어디까지 진행되었는지** 알려줍니다.

***

## 언제 사용하나요?

인덱싱 상태 확인 노드는 **지식 추가 노드와 함께 사용**합니다.\
지식 추가 후 바로 해당 지식을 검색해야 하는 워크플로우에서, 인덱싱 완료 여부를 확인하는 용도입니다.

* 지식 추가 → **인덱싱 상태 확인** → 완료 확인 후 → 지식 검색
* 루프 노드와 함께 사용하여, 인덱싱이 완료될 때까지 반복 확인

{% hint style="info" %}
#### 지식 추가 노드의 "기존 문서에 추가"를 사용한 경우

기존 문서에 추가 옵션으로 내용을 이어붙인 경우, 인덱싱이 즉시 완료됩니다.\
이 경우에는 인덱싱 상태 확인 노드를 사용하지 않아도 됩니다.

인덱싱 상태 확인이 필요한 경우는 **새 문서를 추가**했을 때(지식 추가 노드의 `status`가 `processing`일 때)입니다.
{% endhint %}

***

## 설정 방법

인덱싱 상태 확인 노드를 클릭하면 설정 패널이 열립니다.\
**입력 변수** 영역에 3개의 필드가 있으며, 이 중 하나 이상을 입력해야 합니다.

### 1. 데이터셋 ID (필수 \*)

확인할 **지식의 고유 ID**입니다.\
이 값을 입력하면 해당 지식에 포함된 **모든 문서**의 인덱싱 상태를 확인합니다.\
특정 문서만 확인하고 싶다면 아래의 배치 ID 또는 문서 ID 목록을 함께 사용하세요.

### 2. 배치 ID

지식 추가 노드가 문서를 추가할 때 생성하는 **작업 묶음 ID**입니다.\
같은 시점에 함께 추가된 문서들은 동일한 배치 ID를 가집니다.\
예를 들어 파일 3개를 한 번에 추가하면 3개 모두 같은 배치 ID가 부여되므로, 배치 ID 하나로 3개 문서의 상태를 한 번에 확인할 수 있습니다.

### 3. 문서 ID 목록

확인할 **개별 문서의 ID 목록**입니다.\
여러 문서 ID를 쉼표(,)로 구분하여 입력할 수도 있습니다.

***

### 입력 변수는 어떻게 연결하나요?

세 필드 모두 지식 추가 노드의 출력에서 가져온 값을 사용합니다.

지식 추가 노드는 도구 노드이므로, 결과가 `json` 변수(배열 타입)에 담깁니다.\
배열 안의 개별 필드(`dataset_id`, `batch` 등)는 변수 선택기에서 직접 선택할 수 없기 때문에, **코드 노드로 먼저 값을 꺼내야** 합니다.

지식 추가 노드와 인덱싱 상태 확인 노드 사이에 코드 노드를 배치합니다.

```python
def main(json_result: list) -> dict:
    result = json_result[0]
    return {
        "dataset_id": result["dataset_id"],
        "batch": result["batch"],
        "document_ids": ",".join(result["document_ids"])
    }
```

* 입력: 지식 추가 노드의 `json` 변수를 `json_result`에 연결
* 출력: `dataset_id`, `batch`, `document_ids`가 각각 별도의 변수로 분리됨

> 지식 추가 → **코드 (값 추출)** → 인덱싱 상태 확인

| 인덱싱 상태 확인 입력 | 코드 노드 출력       | 설명                     |
| ------------ | -------------- | ---------------------- |
| 데이터셋 ID      | `dataset_id`   | 지식의 고유 ID              |
| 배치 ID        | `batch`        | 같은 시점에 추가된 문서 묶음 ID    |
| 문서 ID 목록     | `document_ids` | 추가된 문서의 ID (쉼표 구분 문자열) |

가장 간단한 방법은 **데이터셋 ID만 연결**하는 것입니다.\
해당 지식의 모든 문서 상태를 확인할 수 있습니다.

방금 추가한 문서만 정확히 확인하려면 **배치 ID** 또는 **문서 ID 목록**을 함께 연결하세요.

## 출력 변수

| 변수      | 타입             | 설명            |
| ------- | -------------- | ------------- |
| `text`  | String         | 상태 요약 메시지     |
| `files` | Array\[File]   | 도구가 생성한 파일 목록 |
| `json`  | Array\[Object] | 상세 인덱싱 상태 데이터 |

`json` 변수에 상세한 인덱싱 상태 정보가 담깁니다.\
주요 필드는 다음과 같습니다.

```
{
  "status": "processing",
  "dataset_name": "상담 기록",
  "total": 3,
  "completed": 1,
  "failed": 0,
  "processing": 2,
  "message": "Indexing in progress: 1/3 completed, 2 processing.",
  "documents": [
    {
      "name": "상담_20250320.txt",
      "status": "completed",
      "segments": 12
    },
    {
      "name": "상담_20250321.txt",
      "status": "indexing",
      "progress": 80
    }
  ]
}
```

#### 전체 상태 (`status`)

| 상태           | 의미                               |
| ------------ | -------------------------------- |
| `completed`  | 모든 문서의 인덱싱이 완료되었습니다              |
| `processing` | 인덱싱이 진행 중인 문서가 있습니다              |
| `partial`    | 일부 문서는 완료되었지만, 일부는 실패하거나 진행 중입니다 |
| `error`      | 모든 문서의 인덱싱이 실패했습니다               |

#### 개별 문서 상태 (`documents[].status`)

| 상태          | 의미       | 진행률  |
| ----------- | -------- | ---- |
| `waiting`   | 대기 중     | 0%   |
| `parsing`   | 파일 분석 중  | 20%  |
| `cleaning`  | 데이터 정리 중 | 40%  |
| `splitting` | 텍스트 분할 중 | 60%  |
| `indexing`  | 인덱싱 중    | 80%  |
| `completed` | 완료       | 100% |
| `error`     | 실패       | -    |

완료된 문서에는 `segments` (문서가 몇 개의 조각으로 나뉘었는지), 실패한 문서에는 `error` (오류 원인)가 포함됩니다.

## 활용 예시: 인덱싱 완료까지 기다리기

지식 추가 후 바로 검색해야 하는 워크플로우에서는, 루프 노드와 함께 사용하여 인덱싱 완료를 기다릴 수 있습니다.

> 지식 추가 → 코드 (값 추출) → **루프 시작** → 인덱싱 상태 확인 → 조건 분기 (completed이면 루프 종료) → **루프 끝** → 지식 검색 → 답변

조건 노드에서 `status`가 `completed`인지 확인하고, 완료되지 않았으면 루프를 계속 반복합니다.
