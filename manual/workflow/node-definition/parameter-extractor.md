# 변수 추출

LLM 모델이 생성한 결과를 미소에서 지원하는 변수 형식에 맞춰 여러 개의 변수로 나누어 출력하는 노드입니다. string, array(string), object 같은 형태를 지정해 두면, 각 항목에 맞는 값을 구조화된 형태로 받을 수 있습니다.

결과를 하나의 긴 텍스트로 처리하지 않고 목적에 맞게 나누어 사용할 수 있어, 워크플로우를 더 정확하고 효율적으로 구성할 수 있습니다.

***

## 어떤 역할을 하나요?

LLM 노드는 프롬프트에 따라 다양한 형식의 결과를 생성할 수 있지만, 모든 결과가 `text`라는 **하나의 변수에 문자열(String)로 묶여서** 출력됩니다.

예를 들어, LLM 노드에게 이메일을 생성하도록 요청하면 결과는 다음과 같이 나옵니다.

> &#x20;**`text` :** 메일 제목: 회의 일정 안내, 메일 본문: 안녕하세요. 내일 오후 2시에 팀 회의가 예정되어 있습니다. 회의실 A에서 진행되며 관련 자료를 미리 확인해 주세요.

이 경우 제목과 본문이 하나의 문자열에 함께 포함되어 있기 때문에, 워크플로우에서 메일 제목만 따로 사용하거나 메일 본문만 따로 활용하기가 어렵습니다.

실제로 활용하려면 코드 노드를 사용해 문자열을 파싱하거나 구조를 나누는 추가 작업이 필요합니다.

**변수 추출 노드는 이러한 번거로움을 줄여 줍니다.** 미리 변수의 이름과 타입을 정의하면, LLM이 결과를 생성할 때 해당 구조에 맞게 여러 개의 변수로 나누어 출력합니다.

같은 이메일 생성 요청을 변수 추출 노드로 처리하면, 결과가 다음과 같이 분리되어 나옵니다.

| 변수 이름       | 출력 값                                                              |
| ----------- | ----------------------------------------------------------------- |
| `mail_name` | 회의 일정 안내                                                          |
| `mail_body` | 안녕하세요. 내일 오후 2시에 팀 회의가 예정되어 있습니다. 회의실 A에서 진행되며 관련 자료를 미리 확인해 주세요. |

<figure><img src="../../../.gitbook/assets/image (593).png" alt="" width="375"><figcaption></figcaption></figure>

또 다른 예를 들어, 고객 만족도 설문 데이터를 분석하는 경우를 생각해 보겠습니다.

LLM 노드에게 분석을 요청하면 `text`에 다음과 같이 한 줄로 출력됩니다.

> **LLM 노드의 `text` 출력:**\
> 만족도: 만족, 만족요인: 이벤트팬서비스|응원분위기, 페인포인트: 안내티켓|퇴장동선, 고객니즈: 앱 좌석안내 개선과 퇴장 동선 정비, 핵심키워드: 이벤트|완벽한하루|퇴장동선, 요약: 이벤트와 경기 모두 만족하나 안내·퇴장 동선 개선 필요

6개의 인사이트가 추출되었지만, 모두 하나의 문자열 안에 섞여 있어 개별 항목을 활용하기 어렵습니다.

변수 추출 노드를 사용하면 결과가 처음부터 분리됩니다.

| 변수 이름   | 출력 값                           |
| ------- | ------------------------------ |
| `만족도`   | 만족                             |
| `만족요인`  | 이벤트팬서비스\|응원분위기                 |
| `페인포인트` | 안내티켓\|퇴장동선                     |
| `고객니즈`  | 앱 좌석안내 개선과 퇴장 동선 정비            |
| `핵심키워드` | 이벤트\|완벽한하루\|퇴장동선               |
| `요약`    | 이벤트와 경기 모두 만족하나 안내·퇴장 동선 개선 필요 |

별도의 코드 처리 없이도 각 항목을 조건 노드에서 비교하거나, API 요청 노드에서 파라미터로 바로 사용할 수 있습니다.

이처럼 결과가 구조화되어 있으면, 이후 워크플로우에서 각각 필요한 값만 선택해 사용할 수 있습니다.

| 구분    | LLM 노드                        | 변수 추출 노드                          |
| ----- | ----------------------------- | --------------------------------- |
| 출력 방식 | `text` 하나에 문자열로 출력            | 정의한 변수 각각에 분리하여 출력                |
| 출력 타입 | 문자열(String)만 가능               | 텍스트, 숫자, 참/거짓, 선택, 배열 등 다양한 타입 지원 |
| 후처리   | 여러 정보를 분리하려면 코드 노드 등 추가 작업 필요 | 별도 후처리 없이 바로 활용 가능                |
| 용도    | 자유로운 텍스트 생성 (답변, 요약, 번역 등)    | 텍스트에서 정형 데이터 추출                   |

{% hint style="info" %}
LLM 노드는 자유로운 텍스트 생성에 적합하고, 변수 추출 노드는 결과를 구조화하여 이후 노드에서 바로 활용해야 할 때 적합합니다.
{% endhint %}

## 설정 방법

### 1. 모델

추출에 사용할 LLM 모델을 선택합니다. 모델 관련 내용은 LLM 가이드와 동일합니다.

{% content-ref url="llm.md" %}
[llm.md](llm.md)
{% endcontent-ref %}

***

### 2. 입력 변수

LLM에게 전달할 정보가 담긴 변수를 선택합니다.&#x20;

선택된 변수는 프롬프트에 별도로 작성하지 않아도, LLM의 배경지식처럼 자동으로 함께 전달됩니다.

<figure><img src="../../../.gitbook/assets/image (627).png" alt="" width="323"><figcaption></figcaption></figure>

{% hint style="info" %}
#### 여러 개의 정보를 입력하고 싶은데, 입력 변수는 하나뿐인가요?

여러 개의 정보를 입력하고 싶다면, 입력 변수가 하나만 있어도 아래 지시사항에서 직접 여러 값을 작성해 사용할 수 있습니다.

LLM 프롬프트에서와 동일하게 `/`를 입력해 변수를 직접 삽입할 수 있습니다.
{% endhint %}

***

### 3. 추출할 변수 정의

이 노드의 핵심 설정입니다. 어떤 정보를 뽑아낼지 항목별로 하나씩 정의합니다.

* 버튼을 눌러 변수를 추가한 뒤, 각 변수에 대해 아래 항목을 입력합니다.
* 입력한 설명과 설정 정보는 LLM에도 함께 전달되므로, 무엇을 추출해야 하는지 구체적이고 상세하게 적을수록 더 정확한 결과를 얻을 수 있습니다.

<figure><img src="../../../.gitbook/assets/image (629).png" alt="" width="563"><figcaption></figcaption></figure>

| 항목    | 설명                                   | 예시                           |
| ----- | ------------------------------------ | ---------------------------- |
| 이름    | 추출 결과가 담길 변수 이름 (영문, 숫자, `_`만 사용 가능) | `mail_title`, `satisfaction` |
| 타입    | 추출할 데이터의 유형                          | 문자열, 숫자, 선택 등                |
| 설명    | AI에게 무엇을 추출할지 안내하는 문장                | "이메일의 제목 부분을 추출"             |
| 필수 여부 | 반드시 추출되어야 하는 항목인지                    | 필수 / 선택                      |

{% hint style="info" %}
#### 추출 정확도를 높이려면?

*   추출할 변수가 여러 개라면 설명이 모호할수록 원하는 값이 잘못 분리되거나, 다른 변수의 내용이 섞여 추출될 수 있습니다.

    예를 들어 `mail_name`, `mail_body`처럼 변수를 나눠 정의했다면, 각 변수마다 **어떤 내용을**, **어떤 형식으로** 추출해야 하는지 구체적으로 작성해야 합니다.
* 단순히 `제목`, `본문`처럼 짧게 쓰기보다, `이메일의 제목 한 줄만 추출`, `인사말을 포함한 메일 본문 전체를 문자열로 추출`처럼 상세히 적을수록 오류를 줄이고 정확도를 높일 수 있습니다.
{% endhint %}

{% hint style="info" %}
#### 도구에서 가져오기

이미 등록된 도구가 있다면, "도구에서 가져오기" 버튼을 눌러 해당 도구의 매개변수 구조를 그대로 가져올 수 있습니다. 변수 이름, 타입, 설명을 하나씩 입력할 필요 없이, 도구에 정의된 항목이 자동으로 채워집니다.
{% endhint %}

***

### 4. 지시사항

LLM이 추출 시 참고할 추가 지침을 작성하는 영역입니다. 필수는 아니지만, 구체적으로 작성할수록 추출 결과의 품질이 높아집니다.

아래처럼 역할, 작업, 입력 기준, 출력 기준을 구조화해서 작성하면 더 안정적으로 원하는 값을 추출할 수 있습니다.

{% code expandable="true" %}
```
<Role>
스포츠 경기장 관람객 VOC를 감정 톤과 맥락 기반으로 분석하는 전문 애널리스트
</Role>

<Task>
관람객 설문 데이터에서 다음 두 변수를 추출합니다.

1. satisfaction: 관람 경험 만족도 (5단계)
2. summary: VOC 핵심 요약 (40자 이내)
</Task>

<Input>
의견, 상세불만족, 개선방법, 가장불만족, 추천점수, 재방문의사 등 모든 필드를 종합적으로 분석합니다.
텍스트의 감정 표현과 추천점수가 충돌할 경우에는 텍스트를 우선합니다.
</Input>

<Output>
<satisfaction>
- 매우만족: 강한 감탄 또는 추천 표현이 있는 경우
  예) "최고", "완벽", "꼭 가보세요"
- 만족: 전반적으로 긍정적인 표현이 있는 경우
  예) "좋았다", "만족", "즐거웠다"
- 보통: 중립적이거나 긍정·부정이 함께 나타나는 경우
- 불만: 불편, 아쉬움, 개선 필요 표현이 있는 경우
  예) "불편했다", "개선 필요"
- 매우불만: 강한 부정 표현이 있는 경우
  예) "최악", "다시 안 온다"
</satisfaction>

<summary>
- 40자 이내로 작성합니다.
- 핵심 만족 요소 또는 불만 요소를 포함합니다.
  예) 시설, 서비스, 좌석, 동선 등
- 간결하고 자연스러운 문장으로 작성합니다.

예시:
- "좌석 시야 좋고 직원 친절해 만족스러운 관람"
- "화장실 부족과 긴 대기시간으로 불편 경험"
</summary>
</Output>
```
{% endcode %}

***

{% hint style="info" %}
### 활용 팁

출력 타입을 `array(string)`으로 설정하면 여러 값을 리스트 형태로 받을 수 있어 반복문과 함께 활용하기 좋습니다.

예를 들어 뉴스 검색 키워드를 5개 생성하도록 설정하면, 각 키워드가 배열로 반환되고 이를 반복문 노드의 입력 변수로 전달할 수 있습니다. 이후 각 키워드를 기준으로 뉴스 검색을 병렬로 실행할 수 있어, 한 번의 요청으로 다양한 결과를 효율적으로 수집할 수 있습니다.
{% endhint %}
