# 시작

시작 노드는 워크플로우의 **출발점**입니다.\
사용자가 입력한 내용이 가장 먼저 들어오는 곳이며, 이후 모든 흐름은 이 노드에서 시작됩니다.

<figure><img src="../../../.gitbook/assets/image (637).png" alt="" width="292"><figcaption></figcaption></figure>

## 어떤 역할을 하나요?

시작 노드는 **워크플로우의 출발점**입니다.\
사용자나 외부 시스템에서 들어온 입력을 받아, 다음 단계로 전달합니다.

예를 들어,

* 사용자가 질문을 입력하면 → 그 질문이 시작 노드로 들어옵니다
* 파일을 업로드하면 → 파일 정보가 시작 노드로 들어옵니다

이렇게 들어온 정보는 이후 LLM, 조건, 지식 검색 등 다양한 노드로 이어집니다.

***

## 언제 사용하나요?

시작 노드는 **입력 데이터를 정의하는 곳**입니다.\
워크플로우에 들어올 값들을 변수로 설정할 때 사용합니다.

* 워크플로우에는 항상 시작 노드가 필요합니다
* 하나의 워크플로우에는 **단 하나의 시작 노드만 존재합니다**

***

## 설정 방법

<figure><img src="../../../.gitbook/assets/image (638).png" alt="" width="234"><figcaption></figcaption></figure>

### 1. 변수 유형

#### **1) 텍스트/문단**

* 텍스트와 문단은 모두 일반 문자를 입력값으로 받습니다.
  * 텍스트는 짧은 내용을 입력할 때 사용하는 입력 타입입니다. 최대 100자까지 입력할 수 있습니다.
  * 문단은 긴 내용을 입력할 때 사용하는 입력 타입입니다. 최대 100,000자까지 입력할 수 있습니다.
* 두 입력 타입의 차이는 입력 가능한 최대 길이뿐이며, 그 외 기능상의 차이는 없습니다.

#### **2) 목록**

* 목록은 사용자가 미리 지정한 항목 중에서 선택할 수 있도록 하는 입력 타입입니다. UI 상에서는 드롭다운 형태로 표시됩니다.
* 목록은 텍스트/문단과 동일한 문자열 형식으로 값을 받습니다.
* 외부 API를 연동해 데이터를 입력받을 때는 주의가 필요합니다. API로 전달되는 값이 목록에 미리 지정한 항목과 **정확하게 일치하지 않으면 오류가 발생할 수 있습니다.**

#### **3) 숫자**

* 텍스트(문자)나 특수문자는 입력할 수 없으며, 오직 숫자 형식만 허용됩니다.
* 주로 수량, 금액, 점수, 횟수 등 계산이나 비교가 필요한 데이터를 다룰 때 사용됩니다.

#### **4) 단일 파일/다중 파일**

* 단일 파일과 다중 파일은 파일을 입력값으로 받는 입력 타입입니다.
* **입력 방식**
  * 로컬 파일 : 내 컴퓨터에 저장된 파일을 업로드합니다.
  * URL : 업로드된 파일의 다운로드 링크로 설정합니다.
* **파일 유형**
  * 기본적으로 전체가 다 활성화되어있으며 특정항목만 허용하도록 설정할 수 있습니다.
  * 사용자 지정 버튼 클릭 시, 사용자가 직접 허용 확장자를 지정할 수 있습니다.
* 두 입력 타입의 차이는 업로드 가능한 파일 개수뿐이며, 그 외 기능상의 차이는 없습니다.

### 2. 변수 이름

* 시스템 내에서 사용할 변수명을 설정합니다.&#x20;
* 변수명은 영문으로만 입력할 수 있으며, 공백 없이 작성해야 합니다.&#x20;
* 해당 입력 값의 의미를 쉽게 이해할 수 있도록, 직관적이고 명확한 영문 이름으로 설정하는 것을 권장합니다.

### 3. 입력 필드 이름

* MISO에서 제공되는 UI에서 사용자에게 표시되는 변수의 이름입니다.&#x20;
* 변수명처럼 간단하게 작성해도 되고, “입력 언어를 선택해주세요.”와 같이 질문 형태로 작성해도 좋습니다.

<figure><img src="../../../.gitbook/assets/image (641).png" alt="" width="278"><figcaption></figcaption></figure>

### 4. 최대 길이

* 텍스트/문단 유형 변수에 입력할 수 있는 최대 문자 수를 설정합니다.
* 워크플로우 실행 시, “input form must be less than 1 characters” 오류가 발생하는 경우, 설정된 최대 길이보다 입력된 데이터의 길이가 길다는 의미입니다.&#x20;
  * 이 경우 해당 필드의 최대 길이를 늘려야 합니다.
  * 보통 텍스트로 설정되어 있는 경우가 많으므로, 문단 형태로 변경하는 것을 권장합니다.

<figure><img src="../../../.gitbook/assets/image (630).png" alt="" width="320"><figcaption></figcaption></figure>

### 5. 필수 옵션

* 필수 입력은 기본적으로 활성화되어 있습니다.&#x20;
*   만약 해당 입력 변수가 항상 전달되는 값이 아니라면 필수 입력을 비활성화해야 합니다.&#x20;

    * 필수 입력이 활성화된 상태에서 값이 입력되지 않을 경우, 워크플로우는 실행되지 않으며 '필수 입력값을 입력해주세요' 오류가 발생합니다.

## 시스템 변수는 무엇인가요?

* 입력 변수를 따로 지정하지 않아도 시스템에서 기본으로 제공되는 변수입니다.&#x20;

<figure><img src="../../../.gitbook/assets/image (635).png" alt="" width="195"><figcaption></figcaption></figure>

### 1. 사용자 입력

* 대화형 워크플로우에서만 사용 가능한 변수입니다.
* 대화형 워크플로우에서는 채팅 입력창이 기본 UI로 제공되며, 이때:
  * 사용자가 입력한 메시지는 `sys.query`로 전달됩니다.
  * 입력창의 파일 첨부 버튼을 통해 업로드한 파일은 `sys.files`로 전달됩니다.

<figure><img src="../../../.gitbook/assets/image (632).png" alt="" width="399"><figcaption></figcaption></figure>

| 변수        | 설명                 | 활용 예시                |
| --------- | ------------------ | -------------------- |
| sys.query | 사용자가 방금 입력한 메시지    | LLM에서 사용자의 질문, 지식 검색 |
| sys.files | 사용자가 함께 업로드한 파일 목록 | 문서 분석, 이미지 처리        |

* **\[주의] 일반 워크플로우에서는 해당 변수를 사용할 수 없습니다.**
  * 일반 워크플로우에서는 별도로 정의한 입력 변수(문단, 파일 등)를 사용해야 합니다.

### 2. 사용자 정보

* 현재 이 앱을 사용하는 사용자의 정보를 담고 있는 변수들입니다.&#x20;
* 환영 메시지, 사용자에게 메일 전송 등에서 많이 사용됩니다.

| 변수              | 설명         | 활용 예시   |
| --------------- | ---------- | ------- |
| sys.user\_name  | 사용자의 이름    | 개인화된 인사 |
| sys.user\_email | 사용자의 이메일   | 메일 전송   |
| sys.user\_id    | 사용자의 고유 ID | 사용자별 처리 |

### 3. 대화 맥락

현재 대화의 정보를 담고 있으며, 채팅 옵션이 활성화되어 있을 때만 사용할 수 있습니다.

| 변수                   | 설명           | 활용 예시           |
| -------------------- | ------------ | --------------- |
| sys.conversation\_id | 현재 대화의 고유 ID | API 연동 시        |
| sys.dialogue\_count  | 누적 메시지 수     | 메세지 수에 따른 조건 분기 |

### 4. 시간

* 이전에는 현재 시간을 별도 도구로 조회해야 했지만, 이제는 시스템 변수를 통해 바로 확인할 수 있습니다.&#x20;
* 검색 시 현재 시간을 기준으로 활용하는 등 다양한 상황에서 유용하게 사용할 수 있습니다.

| 변수           | 설명               | 활용 예시    |
| ------------ | ---------------- | -------- |
| sys.datetime | 현재 날짜 및 시간 (KST) | 시간 기반 응답 |
