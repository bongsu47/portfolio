# 문서 추출기

문서 추출기 노드는 업로드한 문서 파일에서 텍스트만 뽑아주는 노드입니다.

<figure><img src="../../../.gitbook/assets/image (11).png" alt="" width="293"><figcaption></figcaption></figure>

LLM은 PDF나 Word 파일 자체를 그대로 이해하는 것이 아니라, 그 안에 들어 있는 **텍스트 내용**을 기반으로 작업을 수행합니다. 그래서 문서를 그대로 전달하는 것이 아니라, 먼저 텍스트로 변환해주는 과정이 꼭 필요합니다. 이때 사용하는 노드가 바로 문서 추출기입니다.

즉, 문서 파일을 LLM이 이해할 수 있는 형태로 바꿔주는 **중간 단계**라고 생각하시면 됩니다.

***

## 어떤 역할을 하나요?

문서 추출기 노드는 파일의 내용을 읽어서 텍스트로 변환합니다.

예를 들어,

* 사용자가 PDF 보고서를 업로드하면 → 보고서의 텍스트 내용을 추출합니다
* Excel 파일을 업로드하면 → 표 형태의 데이터를 마크다운 테이블로 변환합니다

추출된 텍스트는 이후 LLM 노드에서 분석하거나, 코드 노드에서 가공하는 데 활용할 수 있습니다.

{% hint style="info" %}
#### 문서 추출기 vs 지식

* **문서 추출기**: 워크플로우 실행 중 사용자가 업로드한 파일의 내용을 즉시 추출합니다. 매번 다른 파일을 처리할 때 적합합니다.
* **지식**: 미리 문서를 등록하여 인덱싱합니다. 고정된 참고 자료를 반복적으로 검색할 때 적합합니다.

사용자가 매번 다른 문서를 업로드하여 분석하는 서비스라면 문서 추출기를, 같은 매뉴얼을 반복적으로 참조하는 서비스라면 지식 등록을 사용하세요.
{% endhint %}

***

## 설정 방법

<figure><img src="../../../.gitbook/assets/image (12).png" alt="" width="368"><figcaption></figcaption></figure>

### 1. 입력 파일 선택

텍스트를 추출할 파일 변수를 선택합니다. 단일 파일 또는 파일 배열(여러 파일) 모두 사용할 수 있습니다.

보통 시작 노드의 파일 입력 변수나, `sys.files`(사용자가 업로드한 파일)를 선택합니다.

***

### 2. 원본 형식 유지 (선택)

문서에 포함된 HTML 서식을 유지할지 선택합니다.

| 설정        | 동작                                            |
| --------- | --------------------------------------------- |
| 비활성화 (기본) | HTML 태그를 제거하고 깨끗한 텍스트만 추출합니다                  |
| 활성화       | HTML 서식(`<b>`, `<table>`, `<a>` 등)을 그대로 유지합니다 |

예를 들어, HTML 파일에 아래와 같은 내용이 있다면:

```html
<h1>공지사항</h1>
<p>다음 주 <b>월요일</b>에 회의가 있습니다.</p>
<a href="https://example.com">참고 링크</a>
```

* **비활성화**: `공지사항 다음 주 월요일에 회의가 있습니다. 참고 링크`
* **활성화**: HTML 태그가 포함된 원본 그대로 추출

일반적으로는 비활성화(기본값)로 두는 것이 좋습니다. LLM에 전달할 때는 깨끗한 텍스트가 더 효과적입니다. HTML 구조 자체를 분석하거나 그대로 보여줘야 하는 경우에만 활성화하세요.

## 지원하는 파일 형식

| 분류     | 형식                       | 추출 결과                              |
| ------ | ------------------------ | ---------------------------------- |
| 문서     | PDF                      | 페이지별로 구분된 텍스트 (`--- Page 1 ---` 등) |
|        | Word (.docx)             | 문단 텍스트 + 표는 마크다운 테이블로 변환           |
|        | PowerPoint (.pptx)       | 슬라이드 내 텍스트와 표 내용                   |
| 스프레드시트 | Excel (.xlsx)            | 시트별로 구분된 마크다운 테이블                  |
|        | CSV                      | 마크다운 테이블 형태                        |
| 텍스트    | TXT, Markdown, HTML, XML | 원본 텍스트 그대로                         |
| 데이터    | JSON                     | 보기 좋게 정렬된 JSON                     |
|        | YAML                     | 구조 유지한 텍스트                         |
| 이메일    | EML, MSG                 | 제목, 발신자/수신자/참조/숨은참조, 본문            |
| 전자책    | EPUB                     | 챕터별 텍스트                            |
| 자막     | VTT                      | 자막 텍스트                             |

{% hint style="info" %}
#### 지원되지 않는 형식

* PowerPoint (.ppt)는 지원되지 않습니다. .pptx 형식으로 변환 후 사용해주세요.
* 비밀번호가 설정된 파일은 추출할 수 없습니다.
* 레거시 Word (.doc)나 Excel (.xls)보다는 최신 형식(.docx, .xlsx)을 사용하는 것을 권장합니다.
{% endhint %}

***

## 표나 이미지도 추출되나요?

회의록처럼 텍스트 중심 문서는 단순 추출만으로도 충분히 처리할 수 있습니다.

Word와 Excel에 포함된 **표**는 마크다운 테이블로 변환되어 추출됩니다. 다만 **이미지**는 추출되지 않으며, 이미지에 포함된 텍스트(스캔 문서 등)도 인식되지 않습니다.

이미지나 표가 포함된 복잡한 레이아웃 문서는 어떤 영역이 표/이미지인지 먼저 구분한 뒤, 요소별로 알맞은 방식으로 처리해야 합니다. 이 내용은 이후 예제에서 다뤄보겠습니다.

***

## 출력 변수

| 변수     | 설명                                            |
| ------ | --------------------------------------------- |
| `text` | 추출된 텍스트. 단일 파일 입력 시 텍스트 하나, 여러 파일 입력 시 텍스트 배열 |

여러 파일을 한꺼번에 추출하면, 입력 순서와 동일한 순서로 결과가 반환됩니다.
