# 지식 검색

지식 검색 노드는 등록된 지식에서 관련 정보를 찾아오는 노드입니다.

사용자의 질문과 관련된 내용을 지식에서 검색하여, 이후 LLM이 정확한 답변을 생성할 수 있도록 참고 자료를 제공합니다. AI가 학습하지 않은 내부 정보, 최신 정보를 활용하려면 이 노드가 필수입니다.

<figure><img src="../../../.gitbook/assets/image (39).png" alt="" width="288"><figcaption></figcaption></figure>

***

## 어떤 역할을 하나요?

지식 검색 노드는 질문의 의미를 이해하여 관련성 높은 내용을 지식에서 찾아줍니다. 단순 키워드 검색이 아니라, 문맥을 파악한 의미 기반 검색을 수행합니다.

예를 들어,

* "반품 절차가 어떻게 되나요?"라고 질문하면 → 반품 관련 매뉴얼에서 관련 내용을 찾아옵니다
* "회의실 예약 방법"을 질문하면 → 사내 규정 문서에서 예약 절차를 찾아옵니다

찾아온 정보는 보통 LLM 노드의 컨텍스트로 연결하여, AI가 정확한 답변을 만드는 데 활용합니다.

{% hint style="info" %}
#### 지식 검색 → LLM 연결이 가장 일반적인 패턴입니다

지식 검색 노드 단독으로는 "검색 결과 목록"만 반환합니다. 이 결과를 사용자에게 자연스러운 문장으로 전달하려면 LLM 노드의 컨텍스트에 연결해야 합니다. 지식 검색 + LLM 조합은 워크플로우에서 가장 많이 사용되는 기본 패턴입니다.
{% endhint %}

***

## 설정 방법

지식 검색 노드를 클릭하면 설정 패널이 열립니다.

<figure><img src="../../../.gitbook/assets/image (663).png" alt="" width="331"><figcaption></figcaption></figure>

### 1. 입력 변수

패널 상단의 **입력 변수** 영역에서 검색에 사용할 질문이 담긴 변수를 선택합니다.\
클릭하면 선택 가능한 변수 목록이 나타납니다.

* 가장 일반적인 선택은 `sys.query` (사용자가 입력한 원래 질문)입니다.
* LLM 노드를 활용해 사용자의 질문을 더 명확하게 정제하면, 검색 정확도를 더욱 높일 수 있습니다.

***

### 2. 지식 선택

**지식** 영역에서 검색 대상 지식을 선택합니다.\
"지식 검색" 버튼을 클릭하면 현재 워크스페이스에 등록된 지식 목록이 나타나며, 하나 이상 선택할 수 있습니다.

선택한 지식에는 다음과 같은 태그가 표시됩니다.

| 태그   | 의미                          |
| ---- | --------------------------- |
| 공유받음 | 다른 사용자가 만든 지식을 공유받아 사용 중    |
| 리랭크  | 현재 이 지식에 리랭킹 검색 방식이 적용되어 있음 |

{% hint style="info" %}
#### 한 노드에 여러 지식을 넣을까요, 노드를 나눌까요?

검색할 지식이 여러 개일 때 두 가지 방법이 있습니다.

**방법 1. 하나의 지식 검색 노드에 여러 지식을 넣기**\
모든 지식의 검색 결과가 한곳에 모이고, 통합 Top K로 상위 결과만 추려냅니다.\
예를 들어 지식 A에서 5건, 지식 B에서 5건이 나와도 통합 Top K가 4이면 전체에서 관련도 높은 4건만 반환됩니다.

**방법 2. 지식마다 별도의 지식 검색 노드를 두기**\
각 노드가 독립적으로 검색하므로, 각각의 Top K만큼 결과를 가져옵니다.\
지식 A에서 3건, 지식 B에서 3건을 확실하게 확보할 수 있습니다.
{% endhint %}

<figure><img src="../../../.gitbook/assets/image (33).png" alt="" width="563"><figcaption></figcaption></figure>

|        | 한 노드에 여러 지식                | 노드를 나눠서 각각 검색                |
| ------ | -------------------------- | ---------------------------- |
| 검색 결과  | 통합 Top K로 전체에서 상위 N건       | 지식마다 각각 Top K건 확보            |
| 장점     | 설정이 간단하고, 리랭크 모델로 통합 정렬 가능 | 특정 지식의 결과가 묻히지 않음            |
| 적합한 경우 | 비슷한 주제의 지식을 함께 검색할 때       | 주제가 다른 지식에서 각각 반드시 결과가 필요할 때 |

{% hint style="info" %}
예를 들어, "인사 규정"과 "제품 매뉴얼"처럼 주제가 전혀 다른 지식을 한 노드에 넣으면, 한쪽 지식의 결과가 다른 쪽에 밀려 아예 나오지 않을 수 있습니다. 이런 경우 노드를 나누는 것이 안전합니다.
{% endhint %}

***

### 3. 개별 지식의 검색 설정

선택한 지식 이름 옆의 `∨` 버튼을 누르면 해당 지식의 검색 설정이 펼쳐집니다.

각 지식마다 검색 방식을 다르게 설정할 수 있으며, 기본적으로는 지식 업로드 시 설정한 값을 따르지만 여기서 개별적으로 변경할 수도 있습니다.

<figure><img src="../../../.gitbook/assets/image (35).png" alt="" width="326"><figcaption></figcaption></figure>

#### **검색 방법**

4개의 탭 중 하나를 선택합니다.

| 탭             | 설명                                                |
| ------------- | ------------------------------------------------- |
| **유사도**       | 질문의 의미를 이해하여 유사한 내용을 검색합니다. 가장 기본적인 방식입니다.        |
| **키워드**       | 정확한 단어 일치로 검색합니다. 고유명사, 코드명, 제품 번호 등을 찾을 때 적합합니다. |
| **하이브리드 가중치** | 유사도 검색과 키워드 검색을 동시에 수행한 뒤, 두 점수를 비율로 합산합니다.       |
| **하이브리드 리랭크** | 유사도 검색과 키워드 검색을 동시에 수행한 뒤, 리랭킹 모델이 결과를 재정렬합니다.    |

{% hint style="info" %}
#### 하이브리드 가중치와 하이브리드 리랭크의 차이

두 방식 모두 유사도 검색과 키워드 검색을 함께 사용합니다. 차이는 **두 검색 결과를 합치는 방법**입니다.

**하이브리드 가중치**는 유사도 검색 점수와 키워드 검색 점수를 **정해진 비율로 더하여** 최종 점수를 계산합니다.

> 예: 유사도 가중치 0.7 + 키워드 가중치 0.3\
> 최종 점수 = (유사도 점수 x 0.7) + (키워드 점수 x 0.3)

비율을 직접 조절할 수 있어 결과를 예측하기 쉽고, 추가 모델 호출이 없어 빠릅니다.

**하이브리드 리랭크**는 두 검색 결과를 모은 뒤, **리랭킹 모델이 질문과 각 결과를 다시 읽고 관련성을 새로 평가**합니다.

> 예: 유사도 검색에서 5건 + 키워드 검색에서 5건 = 후보 10건\
> → 리랭킹 모델이 10건을 모두 다시 읽고 관련성 순으로 재정렬

AI가 직접 판단하므로 정확도가 더 높지만, 리랭킹 모델을 추가로 호출하기 때문에 처리 시간과 비용이 증가합니다.
{% endhint %}

#### **Top K**

검색 결과를 최대 몇 건까지 가져올지 설정합니다.\
숫자를 직접 입력하거나 슬라이더로 조절합니다. 기본값은 **3**입니다.

너무 적으면 관련 정보를 놓칠 수 있고, 너무 많으면 관련 없는 내용이 섞여 AI의 답변 품질이 떨어질 수 있습니다.

#### **점수 임계값**

토글을 켜면 활성화됩니다. 관련도 점수가 이 값 이상인 결과만 가져옵니다.\
예를 들어 0.5로 설정하면, 관련도가 0.5 미만인 결과는 자동으로 제외됩니다.

#### **리랭크 모델**

**하이브리드 리랭크** 탭을 선택하면 나타나는 설정입니다.\
드롭다운을 클릭하여 사용할 리랭킹 모델을 선택합니다.

{% hint style="info" %}
#### 리랭킹 모델이란?

검색 결과의 순서를 **다시 매기는** AI 모델입니다.\
1차 검색으로 가져온 후보 문서들을 리랭킹 모델이 다시 읽고, 질문과의 관련성을 정밀하게 평가하여 순서를 재정렬합니다.

현재 지원되는 리랭킹 모델은 다음과 같습니다.
{% endhint %}

***

#### 4. 여러 지식을 선택한 경우 (다중 지식 설정)

지식을 2개 이상 선택하면, 개별 지식 설정 아래에 **전체 검색 결과에 대한 통합 설정**이 나타납니다.

<figure><img src="../../../.gitbook/assets/image (34).png" alt="" width="331"><figcaption></figcaption></figure>

**Top K**

모든 지식의 검색 결과를 합산한 뒤, 최종적으로 가져올 최대 건수입니다.\
예를 들어 지식 A에서 4건, 지식 B에서 4건이 나왔더라도 Top K가 4이면 관련도가 높은 상위 4건만 반환됩니다.

**점수 임계값**

토글을 켜면 활성화됩니다. 여러 지식에서 모인 결과 중 이 점수 이상인 것만 최종 반환합니다.

**리랭크 모델**

여러 지식의 검색 결과를 통합하여 재정렬할 리랭킹 모델을 선택합니다.\
서로 다른 임베딩 모델로 등록된 지식들의 결과를 하나의 기준으로 정렬할 때 특히 유용합니다.

{% hint style="info" %}
리랭킹 모델은 선택 사항입니다. 설정하지 않으면 각 지식의 검색 점수 기준으로 정렬됩니다.
{% endhint %}

***

#### 5. 테이블 검색

{% hint style="warning" %}
테이블 검색은 **실험실 기능**입니다. 사용하려면 관리자가 실험실 설정에서 테이블 검색을 먼저 활성화해야 합니다.
{% endhint %}

Excel(.xlsx), CSV(.csv) 등 표 형식의 데이터를 등록한 지식에서 사용할 수 있는 검색 방식입니다.\
일반적인 유사도 검색과 달리, 표의 행과 열 구조를 이해하여 정확한 데이터를 조회합니다.

예를 들어, 제품 가격표가 등록된 지식에서 "A제품의 가격이 얼마야?"라고 물으면, AI가 표에서 해당 행을 찾아 정확한 가격을 반환합니다.

실험실에서 기능을 활성화하면, 표 형식 지식의 검색 방법 탭에 **테이블** 탭이 추가로 나타납니다.

***

## 출력 변수

| 변수       | 타입             | 설명           |
| -------- | -------------- | ------------ |
| `result` | Array\[Object] | 검색된 문서 조각 목록 |

`result`는 검색된 문서 조각들이 **관련도 점수가 높은 순서대로** 담긴 배열입니다.\
각 항목은 다음과 같은 구조를 가집니다.

```
[
  {
    "title": "휴가 및 근태 관리 규정",
    "content": "연차 신청은 최소 3일 전에 해야 하며, 팀장 승인 후 확정됩니다. 반차는 당일 오전 9시 이전에 신청할 수 있습니다.",
    "metadata": {
      "dataset_name": "인사 규정",
      "document_name": "휴가 및 근태 관리 규정",
      "score": 0.92,
      "position": 1
    }
  },
  {
    "title": "휴가 및 근태 관리 규정",
    "content": "연차는 입사일 기준으로 매년 15일이 부여되며, 미사용 연차는 다음 해로 이월되지 않습니다.",
    "metadata": {
      "dataset_name": "인사 규정",
      "document_name": "휴가 및 근태 관리 규정",
      "score": 0.85,
      "position": 2
    }
  }
]
```

| 필드                       | 설명                        |
| ------------------------ | ------------------------- |
| `title`                  | 검색된 문서의 제목                |
| `content`                | 검색된 내용 (문서에서 질문과 관련된 부분)  |
| `metadata.dataset_name`  | 이 결과가 속한 지식의 이름           |
| `metadata.document_name` | 이 결과가 속한 원본 문서의 파일명       |
| `metadata.score`         | 관련도 점수 (1에 가까울수록 관련도가 높음) |
| `metadata.position`      | 정렬 순서 (1이 가장 관련도가 높은 결과)  |

이 결과는 보통 LLM 노드의 컨텍스트에 연결하여 사용합니다.\
LLM이 검색된 내용을 참고하여 사용자에게 자연스러운 문장으로 답변을 생성합니다.\
템플릿 노드에 연결하면 검색 결과를 원하는 형식으로 가공할 수도 있습니다.

***

## 검색 정확도를 높이는 팁

기본 설정만으로 원하는 결과가 나오지 않을 때, 워크플로우 구성을 바꾸면 검색 정확도를 크게 개선할 수 있습니다.

#### 1. 검색 전에 LLM으로 질문 다듬기

사용자의 질문을 그대로 검색하면 정확도가 떨어질 수 있습니다.\
예를 들어 "그거 어떻게 해?"처럼 맥락이 부족한 질문은 좋은 검색 결과를 얻기 어렵습니다.

지식 검색 노드 앞에 LLM 노드를 배치하여 질문을 정제하면 검색 품질이 올라갑니다.

> 시작 → **LLM (질문 정제)** → 지식 검색 → LLM (답변 생성) → 답변

LLM에게 "사용자의 질문을 검색에 적합한 형태로 다시 작성해줘"라고 지시하면 됩니다.\
"그거 어떻게 해?" → "연차 신청 절차와 방법"처럼 명확한 검색어로 변환됩니다.

***

#### 2. 가상 문서를 만들어 검색하기 (HyDE)

일반적인 검색은 **질문 → 문서**를 비교합니다.\
하지만 질문과 문서는 문체가 다르기 때문에, 의미가 같아도 유사도 점수가 낮게 나올 수 있습니다.

이 문제를 해결하는 방법이 HyDE(Hypothetical Document Embeddings)입니다.\
LLM에게 "이 질문에 대한 답변을 추측해서 작성해줘"라고 요청하여 **가상의 답변 문서**를 만든 뒤, 그 가상 문서로 검색합니다.

> 시작 → **LLM (가상 답변 생성)** → 지식 검색 → LLM (답변 생성) → 답변

**문서 → 문서**를 비교하게 되므로 유사도가 높아져 검색 정확도가 올라갑니다.\
다만 LLM을 한 번 더 호출하므로 처리 시간과 비용이 증가합니다.

***

#### 3. 여러 검색어를 만들어 병렬로 검색하기

하나의 질문에서 여러 관점의 검색어를 생성하여 각각 검색하면, 다양한 관련 문서를 놓치지 않을 수 있습니다.

LLM에게 "이 질문을 3가지 다른 표현으로 바꿔줘"라고 요청한 뒤, 각 표현으로 지식 검색 노드를 병렬 실행합니다.

> 시작 → **LLM (검색어 3개 생성)** → 코드 노드 (분리) → **지식 검색 x 3 (병렬)** → 결과 합산 → LLM (답변 생성) → 답변

예를 들어, "반품 절차가 어떻게 되나요?"라는 질문에서 다음과 같은 검색어가 생성됩니다.

* "반품 절차 및 단계"
* "상품 반환 신청 방법"
* "환불 처리 프로세스"

하나의 검색어로는 찾지 못했던 문서를 다른 검색어로 찾을 수 있어, 검색 범위가 넓어집니다.

***

#### 4. 검색 결과를 LLM으로 한번 더 걸러내기

검색 결과에 관련 없는 내용이 섞여 있으면 AI 답변의 품질이 떨어집니다.\
지식 검색 후 LLM 노드를 추가하여 "검색 결과 중 질문과 관련 있는 내용만 골라줘"라고 지시하면, 불필요한 결과를 제거할 수 있습니다.

> 시작 → 지식 검색 → **LLM (결과 필터링)** → LLM (답변 생성) → 답변

Top K를 넉넉하게 설정하여 후보를 많이 가져온 뒤, LLM이 관련 있는 내용만 골라내는 방식입니다.

***
