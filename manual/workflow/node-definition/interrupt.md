# 인터럽트

{% hint style="warning" %}
#### 사용 전 확인사항

* **실험실 기능**: 인터럽트 노드는 실험실 기능입니다. 사용하려면 관리자가 해당 기능을 활성화해야 합니다.
* **챗플로우 전용**: 인터럽트 노드는 챗플로우에서만 사용할 수 있습니다. 일반 워크플로우(Completion 모드)에서는 노드 목록에 표시되지 않습니다.
{% endhint %}

인터럽트 노드는 워크플로우 실행을 **일시 정지**하고, 사용자의 입력을 기다린 뒤 다시 이어서 실행하는 노드입니다.

<figure><img src="../../../.gitbook/assets/image (23).png" alt="" width="296"><figcaption></figcaption></figure>

워크플로우가 자동으로 쭉 실행되는 것이 아니라, 중간에 사람의 판단이나 추가 정보가 필요한 경우에 사용합니다. 사용자가 응답을 제출하면 워크플로우가 다시 재개됩니다.

***

## 어떤 역할을 하나요?

인터럽트 노드는 워크플로우 도중에 사용자에게 질문을 하거나 선택을 요청합니다.

<figure><img src="../../../.gitbook/assets/image (22).png" alt="" width="441"><figcaption></figcaption></figure>

예를 들어,

* LLM이 작성한 초안을 사용자에게 보여주고 → **승인** 또는 **거절**을 받습니다
* 추가 정보가 필요할 때 → **폼 입력**으로 사용자에게 데이터를 요청합니다
* 여러 처리 방식 중 하나를 선택해야 할 때 → **분기 선택**으로 사용자가 경로를 결정합니다

조건 노드가 규칙에 따라 자동 분기한다면, 인터럽트 노드는 **사람의 판단**에 따라 분기합니다.

{% hint style="info" %}
#### 인터럽트 vs 조건 노드

* **인터럽트**: 워크플로우를 멈추고 사용자의 응답을 기다립니다. 사람의 판단이 필요할 때 적합합니다.
* **조건**: 변수 값을 자동으로 비교하여 분기합니다. 미리 정해진 규칙으로 처리할 때 적합합니다.

승인 절차, 사용자 확인, 추가 정보 입력 등 사람의 개입이 필요한 경우에만 인터럽트를 사용하세요.
{% endhint %}

***

## 인터럽트 유형

인터럽트 노드는 세 가지 유형을 지원합니다. 설정 패널에서 유형을 선택할 수 있습니다.

### 1. 승인

<figure><img src="../../../.gitbook/assets/image (24).png" alt="" width="563"><figcaption></figcaption></figure>

사용자에게 **승인** 또는 **거절**을 요청합니다.

* 워크플로우가 일시 정지되면 채팅 화면에 승인/거절 버튼이 표시됩니다
* 승인을 누르면 "승인" 경로로, 거절을 누르면 "거절" 경로로 워크플로우가 이어집니다
* 캔버스에서도 승인/거절 두 개의 출력 핸들이 표시되어, 각 경로에 다른 노드를 연결할 수 있습니다

| 출력 변수      | 타입      | 설명                 |
| ---------- | ------- | ------------------ |
| `approved` | Boolean | 승인 여부 (true/false) |
| `comment`  | String  | 사용자가 남긴 코멘트 (선택)   |

***

### 2. 폼 입력

<figure><img src="../../../.gitbook/assets/image (25).png" alt="" width="563"><figcaption></figcaption></figure>

사용자에게 **폼(양식)**&#xC744; 보여주고 데이터를 입력받습니다.

* 설정 패널에서 필드를 추가하고, 각 필드의 변수명/타입/필수 여부를 지정합니다
* 워크플로우가 일시 정지되면 채팅 화면에 폼이 표시됩니다
* 사용자가 폼을 작성하고 제출하면 워크플로우가 이어집니다

#### **지원하는 필드 타입**

| 타입     | 설명           | 입력 UI     |
| ------ | ------------ | --------- |
| 텍스트    | 한 줄 텍스트      | 텍스트 입력란   |
| 텍스트 영역 | 여러 줄 텍스트     | 텍스트 영역    |
| 숫자     | 숫자 값         | 숫자 입력란    |
| 참/거짓   | Boolean 값    | 토글 탭      |
| 단일 선택  | 옵션 중 하나 선택   | 라디오/드롭다운  |
| 다중 선택  | 옵션 중 여러 개 선택 | 체크박스/콤보박스 |

각 필드에는 변수명, 라벨, 설명, 필수 여부를 설정할 수 있습니다. 단일 선택과 다중 선택 필드는 옵션 목록도 함께 설정합니다.

***

### 3. 분기 선택

<figure><img src="../../../.gitbook/assets/image (26).png" alt=""><figcaption></figcaption></figure>

사용자에게 여러 선택지를 보여주고, 하나를 고르게 합니다.

* 설정 패널에서 분기 옵션을 추가합니다 (최소 2개)
* 각 옵션에 라벨을 지정합니다
* 워크플로우가 일시 정지되면 채팅 화면에 선택지 목록이 표시됩니다
* 사용자가 선택한 분기 방향으로 워크플로우가 이어집니다
* 캔버스에서 각 분기 옵션마다 출력 핸들이 표시되어, 서로 다른 노드를 연결할 수 있습니다

| 출력 변수             | 타입     | 설명         |
| ----------------- | ------ | ---------- |
| `selected_branch` | String | 선택된 분기의 ID |

***

## 설정 방법

<figure><img src="../../../.gitbook/assets/image (27).png" alt="" width="458"><figcaption></figcaption></figure>

### 1. 인터럽트 유형 선택

설정 패널 상단에서 **승인**, **폼 입력**, **분기 선택** 중 원하는 유형을 선택합니다.

{% hint style="warning" %}
유형을 변경하면 해당 노드에서 나가는 연결선이 모두 제거됩니다. 유형별로 출력 핸들 구조가 다르기 때문입니다.
{% endhint %}

***

### 2. 안내 메시지

워크플로우가 일시 정지될 때 사용자에게 표시할 안내 메시지를 입력합니다.

예: "아래 내용을 확인하고 승인 또는 거절해주세요."

안내 메시지에는 **변수**를 삽입할 수 있습니다. `{{#노드ID.변수명#}}` 형식으로 이전 노드의 출력 값을 참조할 수 있어, 상황에 맞는 동적 메시지를 만들 수 있습니다.

***

### 3. 폼 필드 (폼 입력 유형만)

폼 입력 유형을 선택하면 폼 필드 설정이 나타납니다.

* **+ 버튼**을 클릭하여 필드를 추가합니다
* 각 필드에 **변수명**(영문, 숫자, 밑줄만 허용), **라벨**, **설명**을 입력합니다
* 필드 타입을 선택하고, 필수 여부를 설정합니다
* 단일 선택/다중 선택 타입은 옵션 목록을 추가합니다 (최대 50자)

***

### 4. 분기 옵션 (분기 선택 유형만)

분기 선택 유형을 선택하면 분기 옵션 설정이 나타납니다.

* **+ 버튼**을 클릭하여 분기를 추가합니다
* 각 분기에 라벨을 입력합니다
* 분기를 삭제하면 해당 분기에 연결된 선도 함께 제거됩니다

***

### 5. 타임아웃

사용자의 응답을 기다리는 최대 시간을 설정합니다.

| 설정   | 범위                  | 기본값    |
| ---- | ------------------- | ------ |
| 타임아웃 | 1분 \~ 1,440분 (24시간) | 1,440분 |

타임아웃 시간이 지나면 워크플로우는 **실패** 상태로 종료됩니다. 슬라이더와 직접 입력 필드로 조절할 수 있습니다.

***

## 출력 변수

| 변수                    | 타입     | 설명          |
| --------------------- | ------ | ----------- |
| `__interrupt_input__` | Object | 사용자의 응답 데이터 |

응답 데이터의 구조는 인터럽트 유형에 따라 달라집니다:

* **승인**: `{ approved: true/false, comment: "..." }`
* **폼 입력**: `{ 변수명1: 값1, 변수명2: 값2, ... }`
* **분기 선택**: `{ selected_branch: "선택된_분기_ID" }`

이후 노드에서 `__interrupt_input__` 변수를 참조하여 사용자 응답을 활용할 수 있습니다.

***

### 제한 사항

| 제한            | 설명                                                   |
| ------------- | ---------------------------------------------------- |
| 챗플로우 전용       | 일반 워크플로우(Completion 모드)에서는 사용할 수 없습니다                |
| 루프/반복 내 사용 불가 | 반복(Iteration) 노드나 루프(Loop) 노드 안에 배치할 수 없습니다          |
| 타임아웃          | 설정한 시간 내에 응답하지 않으면 워크플로우가 실패합니다                      |
| 일시 정지 중 수정    | 워크플로우가 일시 정지된 상태에서 그래프를 수정해도 재개 시에는 정지 시점의 상태로 실행됩니다 |
| 병렬 분기         | 병렬로 실행되는 분기 안에 인터럽트가 있으면 모든 분기가 함께 멈춥니다              |
