# 반복

반복 노드는 **배열**(목록)을 입력으로 받아, 각 항목에 대해 동일한 작업을 자동으로 실행하는 노드입니다.\
순서대로 처리할 수도 있고, 병렬 모드를 켜서 여러 항목을 동시에 처리할 수도 있습니다.

***

## 배열이란?

반복 노드를 이해하려면 먼저 **배열**이 무엇인지 알아야 합니다.

배열은 여러 개의 값을 하나의 목록으로 묶어놓은 것입니다.\
예를 들어, 검색할 키워드가 3개라면 이렇게 하나의 배열로 표현합니다.

```
["AI 트렌드 2025", "생성형 AI 활용 사례", "LLM 비교"]
```

배열 안의 각 값을 **항목**이라고 부릅니다. 위 배열에는 항목이 3개 들어 있습니다.\
워크플로우에서 배열은 주로 코드 노드, 지식 검색, 문서 추출기 등에서 만들어집니다.

반복 노드에 사용할 수 있는 배열 타입은 4가지입니다.

| 타입     | 설명              | 예시                                |
| ------ | --------------- | --------------------------------- |
| 문자열 배열 | 텍스트 목록          | `["사과", "바나나", "딸기"]`             |
| 숫자 배열  | 숫자 목록           | `[85, 92, 78, 95]`                |
| 객체 배열  | 여러 속성을 가진 항목 목록 | `[{name: "김철수", score: 85}, ...]` |
| 파일 배열  | 업로드된 파일 목록      | `sys.files`                       |

배열이 아닌 값(문자열, 숫자 등)은 사용할 수 없습니다.

***

## 어떤 역할을 하나요?

"같은 작업을 여러 데이터에 적용"해야 하는 모든 상황에서 사용합니다.

예를 들어, 사용자의 질문을 분석해서 검색 키워드 5개를 LLM으로 생성했다고 해 보겠습니다.

```
["AI 트렌드", "생성형 AI 활용", "LLM 비교", "프롬프트 엔지니어링", "AI 에이전트"]
```

이 키워드 각각에 대해 웹 검색을 수행하려면, 반복 노드 안에 웹 검색 노드를 하나만 배치하면 됩니다.\
반복 노드가 배열의 각 항목을 꺼내서 웹 검색 노드에 전달하고, 모든 검색 결과를 모아 다시 배열로 출력합니다.

### 병렬 연결과의 차이

워크플로우에서 노드를 병렬로 연결해도 비슷한 작업을 할 수 있지만, 중요한 차이가 있습니다.

노드를 병렬로 연결하면 **워크플로우를 만들 때** 경로의 수가 고정됩니다. 예를 들어, 병렬로 웹 검색 노드를 3개 연결하면 항상 3개의 검색만 수행합니다. 사용자가 어떤 질문을 하든 무조건 3개입니다.

반복 노드는 **배열의 크기에 따라 실행 횟수가 달라집니다.** LLM이 키워드를 3개 생성하면 3번, 7개 생성하면 7번, 10개 생성하면 10번 실행됩니다. 사용자의 요청에 따라 처리 횟수가 유동적으로 바뀌는 것이 핵심입니다.

|        | 병렬 연결                    | 반복 노드                           |
| ------ | ------------------------ | ------------------------------- |
| 실행 횟수  | 워크플로우 설계 시 고정 (예: 항상 3개) | 배열 크기에 따라 유동적 (3개일 수도, 10개일 수도) |
| 적합한 상황 | 처리할 항목이 미리 정해져 있을 때      | 실행 시점에 항목 수가 결정될 때              |
| 설정     | 노드를 직접 여러 개 배치           | 노드 하나만 배치, 배열 입력                |

{% hint style="info" %}
#### 반복 노드 vs 루프 노드

* **반복 노드**: "이 목록의 각 항목을 처리해줘" → 배열의 크기만큼 반복 (횟수가 정해져 있음)
* **루프 노드**: "이 조건이 충족될 때까지 계속해줘" → 조건에 따라 횟수가 달라짐

배열 처리가 필요하면 반복 노드, 조건 기반 반복이 필요하면 루프 노드를 사용하세요.
{% endhint %}

## 설정 방법

<figure><img src="../../../.gitbook/assets/image (650).png" alt=""><figcaption></figcaption></figure>

### 1. 입력 변수

반복 처리할 배열 변수를 선택합니다. **배열 타입 변수만** 선택할 수 있습니다.\
배열이 비어 있으면(`[]`) 내부 노드를 실행하지 않고 빈 배열을 바로 출력합니다.

***

### 2. 반복 내부 노드 구성

반복 노드 안의 + 버튼을 클릭하여 원하는 노드들을 배치합니다. 이 노드들이 배열의 각 항목마다 실행됩니다.\
LLM, 코드, API 요청, 조건 등 대부분의 노드를 내부에 배치할 수 있습니다.

반복 노드 안에서는 두 가지 **특수 변수**를 사용할 수 있습니다.

| 변수      | 설명                 | 예시                                        |
| ------- | ------------------ | ----------------------------------------- |
| `item`  | 현재 처리 중인 항목의 값     | 배열이 `["A", "B", "C"]`일 때, 첫 번째 반복에서 `"A"` |
| `index` | 현재 항목의 순서 (0부터 시작) | 첫 번째: 0, 두 번째: 1, 세 번째: 2                 |

이 변수들은 반복 노드의 변수 목록에서 선택할 수 있습니다.\
예를 들어, LLM 노드의 프롬프트에서 `{{#반복_1.item#}}`으로 현재 항목을 참조합니다.

{% hint style="info" %}
#### index 활용법

`index`는 "지금 몇 번째 항목을 처리 중인지"를 알려주는 변수입니다.

* LLM 프롬프트에서 "문서 \{{#반복\_1.index#\}}번째:"처럼 번호를 매길 때
* 코드 노드에서 첫 번째 항목만 특별하게 처리할 때 (`if index == 0`)
* 진행 상황을 결과에 포함할 때
{% endhint %}

***

### 3. 출력 변수 선택

각 반복에서 저장할 결과 변수를 선택합니다. 반복 내부에 배치한 노드의 출력 중 하나를 선택합니다.\
모든 반복이 끝나면, 각 반복의 결과가 배열로 모여서 출력됩니다.

예를 들어, 3개의 문서를 각각 요약하면:

* 입력: `["문서1 전문...", "문서2 전문...", "문서3 전문..."]`
* 출력 변수로 LLM 노드의 `text`를 선택
* 결과: `["문서1 요약", "문서2 요약", "문서3 요약"]`

***

### 4. 병렬 모드

반복 노드는 **직렬**(순차) 실행이 기본이며, **병렬**(동시) 실행으로 변경할 수 있습니다.

#### **직렬 실행 (기본)**

항목을 **하나씩 순서대로** 처리합니다.

* 첫 번째 항목 처리 완료 → 두 번째 항목 처리 → 세 번째 항목 처리 → ...
* 결과 순서가 입력 순서와 동일하게 보장됩니다.
* 전체 소요 시간 = 각 항목의 처리 시간을 모두 합친 시간

#### **병렬 실행**

여러 항목을 동시에 처리할 수 있으며, 패널에서 **병렬 모드 토글**을 활성화하면 됩니다.

**동시 실행 수(1\~10)**&#xB294; 한 번에 **동시에 처리할 최대 항목 수**를 의미합니다.\
예를 들어 동시 실행 수를 10으로 설정하면, 항목을 **최대 10개씩 나누어 순차적으로 처리**합니다.

예를 들어, 배열에 항목이 30개이고 동시 실행 수를 10으로 설정하면:

* 10개를 동시에 처리 → 완료되면 다음 10개를 동시에 처리 → 마지막 10개 처리
* 전체 30개 항목이 모두 처리됩니다.

배열이 50개, 100개여도 동시 실행 수만큼씩 나누어 모두 처리합니다.

| 항목                | 직렬     | 병렬 (동시 실행 수: 10) |
| ----------------- | ------ | ---------------- |
| 30개의 문서 요약 (각 5초) | 약 150초 | 약 15초 (10개씩 3번)  |

결과 배열은 병렬로 처리하더라도 **항상 입력 순서대로** 정리되어 출력됩니다.

{% hint style="info" %}
#### 언제 병렬을 사용하나요?

**병렬이 적합한 경우:**

* 각 항목의 처리가 서로 독립적일 때 (이전 결과가 다음 항목에 영향을 주지 않음)
* LLM 요약, API 호출, 파일 변환 등 처리 시간이 긴 작업일 때
* 처리 속도가 중요할 때

**직렬이 적합한 경우:**

* 이전 항목의 결과가 다음 항목 처리에 필요할 때
* 외부 API의 호출 횟수 제한(rate limit)이 있을 때
* 처리 순서가 로직에 영향을 줄 때
{% endhint %}

{% hint style="warning" %}
#### 병렬 실행 시 주의사항

병렬 모드에서 반복 내부에 **답변 노드**를 배치하면 주의가 필요합니다.\
여러 항목이 동시에 답변을 생성하면 결과가 뒤섞일 수 있습니다.
{% endhint %}
