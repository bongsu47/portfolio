# 워크플로우 제한사항

워크플로우를 구성할 때 아래 제한 사항이 적용되며 초과 시, 워크플로우를 저장하거나 실행할 수 없습니다.

| 항목             | 제한    |
| -------------- | ----- |
| 최대 노드 개수       | 200개  |
| 트리 깊이 제한       | 50    |
| 병렬 최대 분기 수     | 10개   |
| 병렬 최대 중첩 깊이    | 3단계   |
| 루프 노드 최대 반복 횟수 | 100회  |
| 반복 노드 최대 반복 횟수 | 제한 없음 |

### 최대 노드 개수: 200개

하나의 워크플로우에 배치할 수 있는 노드의 총 개수입니다.\
시작 노드, 끝 노드를 포함한 모든 노드를 합산하며, 200개를 초과하면 노드를 더 추가할 수 없습니다.

### 트리 깊이 제한: 50

시작 노드에서 끝 노드까지의 **가장 긴 경로**에 포함되는 노드 개수입니다.

```
시작 → A → B → C → ... → 끝     ← 경로 1 (깊이 = N)
           └→ D → E → 끝         ← 경로 2 (깊이 = 5)
```

위처럼 분기가 있어 경로가 여러 개라면, 그중 가장 긴 경로를 기준으로 합니다.\
이 예시에서는 경로 1과 경로 2 중 더 긴 쪽이 50을 넘지 않아야 합니다.

워크플로우가 너무 길어지면 처리 시간이 늘어나고 오류 가능성이 높아지므로, 가능하면 흐름을 간결하게 유지하는 것이 좋습니다.

### 병렬 처리 제한

#### 최대 분기 수: 10개

하나의 병렬 노드에서 동시에 나갈 수 있는 분기의 최대 개수입니다.

```
          ┌→ 분기 1
          ├→ 분기 2
병렬 노드 ├→ 분기 3
          ├→ ...
          └→ 분기 10   ← 최대 10개
```

#### 최대 중첩 깊이: 3단계

병렬 노드 안에 또 다른 병렬 노드를 넣을 수 있는 최대 깊이입니다.

```
병렬 노드 A (1단계)
├── 분기 1: LLM 노드
└── 분기 2: 병렬 노드 B (2단계)              ← 허용
            ├── 분기 2-1: 코드 노드
            └── 분기 2-2: 병렬 노드 C (3단계)        ← 허용
                          ├── ...
                          └── 병렬 노드 D (4단계)     ← 제한 초과!
```

3단계까지는 병렬 안에 병렬을 중첩할 수 있지만, 4단계부터는 허용되지 않습니다.

### 반복 제한

| 노드    | 최대 반복 횟수 | 설명                         |
| ----- | -------- | -------------------------- |
| 루프 노드 | 100회     | 조건이 충족되지 않더라도 100회 후 자동 종료 |
| 반복 노드 | 제한 없음    | 입력 목록의 항목 수만큼 반복           |

루프 노드는 무한 반복을 방지하기 위해 최대 100회까지만 반복합니다.

반복 노드는 횟수 제한이 없으며, 입력으로 받은 목록의 항목 수만큼 자동으로 반복합니다.
