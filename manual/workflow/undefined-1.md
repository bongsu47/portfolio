# 워크플로우 구성 및 사용법

## 워크플로우의 구성요소

워크플로우는 노드와 노드 간의 연결을 통해 애플리케이션 동작을 정의합니다.

#### 노드

노드는 애플리케이션 동작을 정의하는 최소 단위입니다. 노드에는 LLM, 지식, 답변 등 MISO에서 제공하는 기본 노드와 도구를 활용하는 노드가 존재합니다.

#### 연결

노드의 실행 흐름을 정의하는 요소입니다. 연결된 노드들을 왼쪽에서 오른쪽 방향으로 실행됩니다.



<figure><img src="../../.gitbook/assets/image (410).png" alt="간단한 워크플로우 구성"><figcaption><p>간단한 워크플로우 예제: 시작 -> LLM -> 종료 순서로 노드가 실행됩니다.</p></figcaption></figure>

## **작업 공간**

애플리케이션의 동작을 정의하는 공간으로 노드를 배치하고 노드간의 연결 작업을 진행합니다. 우클릭 시, 노드 추가와 메모 추가를 진행합니다.

<figure><img src="../../.gitbook/assets/image (453).png" alt="" width="375"><figcaption><p>&#x3C;노드 추가></p></figcaption></figure>

### **노드 추가**

클릭 시, 추가 할 수 있는 노드의 종류와 도구의 종류를 확인 할 수 있습니다. 검색을 통해 원하는 노드를 빠르게 찾을 수 있습니다. 각 노드에 대한 사용 설명은 [링크](node-definition/)를 참조 하세요

### **메모 추가**

작업 공간에 메모를 남길 수 있습니다.

## 퀵 메뉴

<figure><img src="../../.gitbook/assets/image (250).png" alt=""><figcaption><p>&#x3C;퀵 메뉴></p></figcaption></figure>

| 기능     | 설명                                            |
| ------ | --------------------------------------------- |
| 확대/축소  | 편집 화면을 확대/축소합니다. (최소 25% \~ 최대 200%)          |
| 노드 추가  | 편집 화면에 필요한 노드를 추가합니다.                         |
| 메모 추가  | 편집 화면에 메모를 추가합니다.                             |
| 포인터 모드 | 마우스 포인터를 포인터 모드로 변경하며, 노드·메모·연결선을 선택할 수 있습니다. |
| 핸드 모드  | 마우스 포인터를 핸드 모드로 변경하며, 드래그로 화면 이동이 가능합니다.      |
| 노드 정리  | 노드를 보기 좋게 자동 배치합니다.                           |
| 실행 취소  | 이전 작업 상태로 되돌립니다.                              |
| 다시 실행  | 실행 취소된 작업을 다시 적용합니다.                          |

## **도구로 등록**

워크플로우(채팅)를 다른 곳에서 사용할 수 있도록 제공하는 기능입니다. MISO에서는 API 형태의 배포와 Web APP 형식의 배포가 존재 합니다. API는 타 시스템에서 가져다 사용 할 수 있도록 API 키 형태로 제공합니다. WebApp 형식은 URL을 통해 바로 접속 할 수 있도록 제공되고 있으며, MISO는 URL 형식 외에 Iframe 또는 위젯 형식으로 앱을 제공하기도 합니다.

## 앱 조건 설정

<figure><img src="../../.gitbook/assets/image (28).png" alt="" width="440"><figcaption></figcaption></figure>

#### **대화변수**

대화 변수는 채팅 워크플로우(챗플로우) 전용 기능으로, 대화가 진행되는 동안 값이 유지되는 변수입니다.

일반 워크플로우에서는 실행이 끝나면 모든 변수가 사라집니다. 하지만 챗플로우에서는 사용자가 여러 번 메시지를 주고받으므로, 이전 대화에서 저장한 값을 다음 대화에서도 사용할 수 있어야 합니다. 이것이 대화 변수의 역할입니다.

#### **환경변수**

개인 정보 및 자격 증명을 저장하는데 활용할 수 있습니다. Secret 유형으로 지정 시, 앱 내보내기 파일에 포함하지 않습니다.

**+ 추가** 버튼을 클릭시 환경 변수 추가를 위한 팝업이 호출됩니다.

<figure><img src="../../.gitbook/assets/image (117).png" alt=""><figcaption><p>&#x3C;환경 변수 추가></p></figcaption></figure>

환경 변수의 유형은 String, Number, Secret이 있으며, String 유형은 텍스트와 숫자 모두 입력이 가능하지만, Number 유형은 숫자만 입력이 가능합니다. 마지막 Secret 유형은 텍스트와 숫자 모두 입력이 가능하고, 보안을 위한 마스킹 처리 되어 표현됩니다. 추후 앱 내보내기 진행 시, Secret 유형으로 지정된 데이터는 파일에 포함되지 않습니다.&#x20;
