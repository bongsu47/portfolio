# 미소 공유하기

미소로 제작된 앱이나 임베딩된 지식은 공유 시, 제작자로 부터 할당 받은 권한에 따라 이용이 가능 합니다. 이 페이지에서는 공유 범위와 사용자에 대한 할당 정책 및 공유하기 사용에 대한 설명을 합니다.

### **APP 권한 설정**

**사용자**: 미소로 제작된 앱을 실행 하고 앱이 어떻게 구성되어있는지 확인 할 수 있는 권한입니다. 에이전트 앱이라면 프롬프트 및 도구 구성 등에 대한 정보를 확인 할 수 있고, 워크플로우 앱이라면 노드 구성 및 노드 설정을 확인 할 수 있습니다.\
(공유 범위 내에 포함된 유저는 모두 기본적으로 사용자 권한이 부여됩니다)\
단, 앱이 생성된 워크스페이스 외부 사용자는 앱 설정 화면 접근이 불가능합니다.\
\
**편집자**: 미소로 제작 된 앱을 수정하고 사용 현황(로그)를 조회 할 수 있는 권한 입니다. 에이전트 앱이라면 프롬프트 수정 및 도구 구성과 참조할지식에 대한 편집을 진행할 수 있고, 워크플로우 앱이라면 노드에 대한 추가/수정/삭제 등 앱에 대해 편집 할 수 있습니다. 편집자 권한은 현재 워크스페이스에 있는 사용자만 할당이 가능 합니다.\
\
**앱 소유자**: 앱을 소유한 사용자를 말하며, 실행/읽기/쓰기 권한에서 수행하는 일을 모두 할 수 있으며, 앱 설명을 수정하거나 소유권을 타인에게 전달 할 수 있고 앱을 삭제할 수 있습니다.

### **APP 공유 범위 설정**

**비공개**: 앱 생성시, 기본적으로 설정되어 있는 권한으로 앱 소유자만 실행 할 수 있습니다.\
비공개 설정 시, 다른 사용자들에게 접근 권한을 할당 할 수 없습니다.

<figure><img src="../.gitbook/assets/image (682).png" alt=""><figcaption></figcaption></figure>

\
**현재 워크스페이스 공개**: 현재 워크스페이스의 사용자들에게만 앱 사용자 권한을 부여 합니다. 별도의 관리자 승인 없이, 할당이 가능합니다. 현재 워크스페이스에 포함된 사용자에게는 편집자 권한을 부여할 수 있습니다. 사용자 리스트 우측 토글 버튼을 통해 설정할 수 있으며, 좌측 체크박스 선택을 통해 일괄 부여 또는 해제할 수 있습니다.

<figure><img src="../.gitbook/assets/image (683).png" alt=""><figcaption></figcaption></figure>

\
**모든 워크스페이스 공개**: 모든 워크스페이스의 사용자들에게 앱 사용자 권한을 부여 합니다. 모든 워크스페이스 공개를 하기위해서는 관리자에게 승인을 요청하고, 승인이 난 이후 모든 워크스페이스 공개 권한으로 앱 사용이 가능 합니다. 관리자의 승인 전 까지는 이전 공유 범위를 유지 합니다. 외부 워크스페이스 사용자의 상세 리스트는 조회가 불가능하며 편집자 권한을 부여할 수 없습니다.

<figure><img src="../.gitbook/assets/image (684).png" alt=""><figcaption></figcaption></figure>



**외부 공개**: 발행된 앱을 URL을 통해 누구나(비로그인 유저 포함) 실행할 수 있는 권한을 부여합니다. 외부 공개를 하기 위해서는 관리자에게 승인을 요청하고, 승인이 난 이후 외부 공개 권한으로 앱 실행이 가능합니다. 관리자의 승인 전 까지는 이전 공유 범위를 유지 합니다. 외부 공개시에도 편집자 권한은 현재 워크스페이스 사용자에게만 부여가 가능합니다.

<figure><img src="../.gitbook/assets/image (685).png" alt=""><figcaption></figcaption></figure>

외부 공개는 추가로 IP 접근 제한을 할 수 있습니다. 기본은 모든 IP 허용으로 되어 있지만 IP 제한 사용으로 변경하면 아래와 같이 화이트리스트와 블랙리스트를 설정 할 수 있습니다.

<figure><img src="../.gitbook/assets/image (415).png" alt=""><figcaption><p>IP 설정</p></figcaption></figure>

**부분 공개**: 특정 미소 사용자에게 실행 권한을 할당 하기 위한 종류의 공개 입니다. "사용자 추가" 버튼을 클릭하면 권한을 부여할 사용자를 조회할 수 있는 팝업을 통해 사용자 권한을 부여할 수 있습니다. 부분 공개 사용자 추가시에도 편집자 권한은 현재 워크스페이스 사용자에게만 부여가 가능합니다.

공유 요청 이력은  공유 범위 관리 우측 상단 "이력" 버튼을 통해 확인이 가능 합니다.

### **지식 권한 설정**

{% hint style="success" %}
* **지식의 권한과 공유 범위 설정 방식은 기본적으로 APP 권한/공유 설정 방식과 동일합니다.**
{% endhint %}

**사용자**: 해당 지식이 앱에서 참조된 지식으로 사용되었을 때, 정상적으로 지식을 이용할 수 있으며, 해당 지식이 어떠한 문서로 구성되어있는지 확인할 수 있는 권한입니다. 추가로 지식에 대한 청크와 상세 문서 정보를 확인 할 수 있습니다.\
\
**편집자**: 사용자 권한에 추가로 지식에 대한 설정 정보를 수정 하거나, 문서를 추가로 임베딩 할 수 있는 권한 입니다. 해당 권한은 같은 워크스페이스에 소속된 사용자만 할당 할 수 있습니다.



### 지식 공유 범위 설정

**비공개**: 지식 임베딩을 등록한 사용자만 쓸 수 있는 상태 입니다.

**현재 워크스페이스 공개**: 현재 워크스페이스에 해당 지식을 실행 할 수 있는 권한을 부여  합니다. 사용자별로 추가 권한 할당이 되지 않은 상태라면, 앱 실행 시, 연동된 지식으로써 사용만 가능합니다.

**모든 워크스페이스 공개**: 모든 워크스페이스에 해당 지식을 실행 할 수 있는 권한을 부여 합니다. 사용자별로 추가 권한 할당이 되지 않은 상태라면, 앱 실행 시, 연동된 지식으로써 사용만 가능합니다.

**외부 공개**: 지식의 실행 권한을 외부 인원들에게 까지 공유하기 위한 권한 설정 입니다. 외부 공개 앱에서 사용된 참조된 지식은 외부 공개를 맞춰주어야만 앱에서 정상적으로 지식을 사용할 수 있습니다.

**부분 공개**: 특정 사용자들에게만 사용자 권한을 할당 합니다.

{% hint style="success" %}
*   **API를 활용하여 외부 서비스에서 미소 APP을 호출할 경우**

    로그인 여부와 무관하게 서비스 API를 활용하여 미소의 APP과 지식을 호출하는 경우입니다.\
    이 경우 앱과 지식은 모두 로그인 필수 조건을 요하지 않는 **"외부 공개"** 상태일 경우에만 정상 동작합니다.
{% endhint %}

### 앱 & 지식 공유 범위 조합 정책 정리

앱과 지식은 각각 권한을 가지지만, 앱 실행 중 지식을 참조할 때는 **앱 권한과 지식 권한을 둘 다 만족해야 합니다**.\
따라서 앱 공유 범위가 연결된 지식보다 넓어지면, 저장 시점에 아래 둘 중 하나를 선택하게 한다.

1. 지식 권한을 앱 권한에 맞춰 수정한다.
2. 해당 지식을 앱에서 제외한다.

**권한 범위 순서(좁은 권한에서 넓은 권한 순)**

1. 비공개
2. 부분공개
3. 현재 워크스페이스 공개
4. 모든 워크스페이스 공개
5. 외부공개

단, `부분공개 ↔ 부분공개`는 단순히 같은 범위로 보지 않습니다. 대상자가 다를 수 있으므로 앱과 지식의 공유 대상 정합성 확인이 필요합니다.

**실제 동작 기준**

앱이 보인다고 해서 지식까지 항상 참조 가능한 것은 아닙니다.

예를 들어:

* 앱: 현재 워크스페이스 공개
* 지식: 비공개

이 경우 현재 워크스페이스 사용자는 앱을 볼 수 있고 실행할 수 있다. 하지만 지식은 지식 소유자만 읽을 수 있으므로, 일반 사용자의 실행에서는 해당 지식을 참조할 수 없습니다.

이런 상태를 허용하면 사용자는 앱은 실행되는데 답변 품질이 달라지거나 지식 검색 결과가 빠지는 문제가 발생합니다. 그래서 저장 시점에 불일치를 정리해야합니다.

<details>

<summary><strong>앱 &#x26; 지식 공유 범위 케이스 정리</strong></summary>

#### 1. 앱 권한이 지식보다 좁거나 같으면 허용

앱 사용자가 지식 사용 가능 범위보다 적거나 같으면 문제 없다.

예:

* 앱 비공개 + 지식 모든 워크스페이스 공개
* 앱 현재 워크스페이스 공개 + 지식 모든 워크스페이스 공개
* 앱 모든 워크스페이스 공개 + 지식 외부공개

#### 2. 앱 권한이 지식보다 넓으면 저장 시 차단

앱을 더 넓게 공유하려는데 연결된 지식이 더 좁으면 그대로 저장하지 않는다.\
사용자는 각 지식에 대해 아래 중 하나를 선택해야 한다.

* 권한 맞추기
* 앱에서 제외

#### 3. 권한 맞추기

지식 소유자가 변경 가능한 지식이면 앱 권한에 맞춰 지식 권한도 함께 변경한다.\
승인이 필요한 권한이면 앱 권한 요청과 지식 권한 요청을 같이 올린다.

#### 4. 앱에서 제외

해당 지식을 앱 설정에서 제거한다.

제거 대상:

* 앱-지식 조인 정보
* agent dataset config
* agent mode 내부 dataset tool 참조
* retriever resource
* workflow graph의 knowledge retrieval node 참조

즉 UI에서만 빠지는 것이 아니라 실제 앱 실행에서도 해당 지식을 사용하지 않게 한다.

#### 5. 부분공개는 대상자 기준 확인 필요

부분공개는 단순히 권한 단계만 같다고 안전하지 않다.

예:

* 앱 부분공개 대상: A, B
* 지식 부분공개 대상: A

이 경우 B는 앱은 실행할 수 있지만 지식은 참조할 수 없다.\
따라서 부분공개끼리도 공유 대상 정합성을 확인해야 한다.
