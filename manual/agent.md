# 에이전트

앱 만들기에서 앱 생성을 통해 에이전트를 생성하고, 앱 편집에 들어가면 아래와 같은 화면이 호출 됩니다.

<figure><img src="../.gitbook/assets/image (455).png" alt=""><figcaption></figcaption></figure>

## 기본 기능

***

**프롬프트**: 에이전트의 동작을 정의하는 곳으로 다양한 프롬프팅 기법을 통해 에이전트가 수행하는 역할을 정의 합니다.

**입력 변수**: 에이전트 실행 시, 사용자로부터 입력 받아야 하는 정보를 변수로 만들어서 사용할 수 있습니다. 입력 변수는 유형에 따라 텍스트, 문단, 숫자, 목록이 있습니다.

**사용 현황**: MISO는 에이전트의 모니터링과 로그 정보를 제공합니다.

**공유**: 공유를 통해 에이전트의 공유 범위를 설정 합니다. 현재 조직 내 사용자만 사용/모든 조직 사용자가 사용/ URL을 통한 공유 가 존재합니다. ( [참고](undefined-2.md) )

**테스트**: 만들어진 에이전트를 바로 테스트 해볼 수 있는 기능입니다.

**미리보기**: 앱이 배포되었을 때의 형태를 사전에 확인할 수 있습니다.

{% hint style="info" %}
#### 항상 미소에서 지원하는 최신 모델이 에이전트 앱에 자동 적용되게 할 수 있나요?

**Latest 모델**을 선택하면 가능합니다.

아래와 같이 모델 공급사의 모델 Family 별로 "Latest" 표기가된 모델을 리스트에서 선택할 수 있습니다.

해당 모델을 선택할 경우 현재 미소 플랫폼에서 공식 지원하고있으며, 해당 워크스페이스에서 사용 가능한 최신 모델이 자동 적용됩니다.

따라서 사용자는 모델 버전 업데이트시 추가적인 설정 작업 없이 항상 최신 모델을 내 앱에 반영할 수 있습니다.

## 저장하기

<div align="left"><figure><img src="../.gitbook/assets/image (454).png" alt="" width="301"><figcaption></figcaption></figure></div>

* **저장하기**: 현재 편집 화면에 적용된 형상을 저장 합니다. 저장하기만 진행할 경우, 발행이 된 상태가 아니기 때문에, 웹 또는 API에 적용이 된 상태는 아닙니다.
* **발행하기**: 현재 편집 화면에 적용된 형상을 웹과 API에 적용하기 위한 기능 입니다.
  *   발행시 버전명과 수정 상세 내용 기록을 통해 발행 버전 히스토리 관리를 용이하게할 수 있습니다.

      <div align="left"><figure><img src="../.gitbook/assets/image (456).png" alt="" width="188"><figcaption></figcaption></figure></div>
* **버전목록**: APP이 발행 되었던 과거 이력을 확인 합니다.
  *   저장 및 발행된 전체 APP 버전 히스토리를 확인할 수 있습니다.

      <div align="left"><figure><img src="../.gitbook/assets/image (457).png" alt="" width="188"><figcaption></figcaption></figure></div>
  * 연필 모양 아이콘을 클릭 하면, 해당 버전의 이름 및 설명을 변경 할 수 있습니다.
    * 원하는 버전을 선택 한 후, 선택 버전 불러오기를 진행하면 선택된 버전의 형상을 화면에 호출 합니다.(아래 참조)
    *

        <figure><img src="../.gitbook/assets/image (395).png" alt=""><figcaption></figcaption></figure>


    * 취소 버튼 클릭 시, 편집 화면으로 돌아갑니다.
    * 이 버전으로 저장 클릭 시, 앱의편집화면이 선택된 버전으로 돌아갑니다.(단, 저장하기만 선택된 상태이며, 웹이나 API로 활용하기 위해서는 발행하기를 진행하셔야 합니다.)
