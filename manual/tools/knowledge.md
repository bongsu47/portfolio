# 지식 도구

## 지식 도구 사용자 매뉴얼

### 개요

지식 도구는 워크플로우 실행 중에 텍스트나 파일을 지식 저장소에 추가할 수 있는 기능입니다. AI가 대화 중 수집한 정보를 자동으로 저장하고, 나중에 검색할 수 있도록 해줍니다.

#### 이런 경우에 사용하세요

* 대화 중 중요한 정보를 자동으로 저장하고 싶을 때
* 사용자가 업로드한 파일을 지식에 추가하고 싶을 때
* 외부에서 가져온 데이터를 지식으로 축적하고 싶을 때
* 문서 인덱싱 진행 상황을 확인하고 싶을 때

#### 제공되는 도구

| 도구                                      | 설명               |
| --------------------------------------- | ---------------- |
| **지식 추가** (add\_knowledgebase)          | 텍스트나 파일을 지식에 추가  |
| **인덱싱 상태 확인** (check\_indexing\_status) | 추가한 문서의 처리 상태 확인 |

***

### 지식 추가 도구

텍스트 내용이나 파일을 지식 저장소에 추가합니다. 기존 지식을 선택하거나 새로운 지식을 생성할 수 있습니다.

#### 설정 항목

**1. 지식 선택**

저장할 대상 지식을 선택합니다.

* **기존 지식 선택**: 드롭다운에서 워크스페이스에 있는 지식 중 하나를 선택
* **새 지식 생성**: "새 지식 생성"을 선택하면 지식 이름 입력 필드가 나타남

**2. 지식 이름**

새 지식을 생성할 때만 입력합니다. 기존 지식을 선택한 경우에는 표시되지 않습니다.

**3. 지식 설명**

지식에 대한 설명을 입력합니다. AI가 어떤 내용을 저장할지 결정할 때 참고합니다.

**4. 텍스트 콘텐츠**

저장할 텍스트 내용을 입력합니다. AI가 대화 중 수집한 정보나 요약 내용을 여기에 작성합니다.

**5. 파일명**

텍스트 콘텐츠를 저장할 때 사용할 파일명입니다. 입력하지 않으면 "content.txt"로 저장됩니다.

**6. 파일 업로드**

문서 파일을 직접 업로드하여 지식에 추가합니다. PDF, TXT, DOCX 등 다양한 형식을 지원합니다.

**7. 기존 문서에 추가**

같은 이름의 문서가 이미 있을 때의 처리 방식을 선택합니다.

| 설정          | 동작            |
| ----------- | ------------- |
| **끔** (기본값) | 새로운 문서로 별도 생성 |
| **켬**       | 기존 문서에 내용 추가  |

#### 사용 예시

**예시 1: 회의 내용 자동 저장**

```
지식 선택: "프로젝트 회의록"
지식 설명: "주간 회의에서 논의된 내용"
텍스트 콘텐츠: [AI가 요약한 회의 내용]
파일명: "2024-12-28-회의록.txt"
```

**예시 2: 새 지식 생성하여 저장**

```
지식 선택: "새 지식 생성"
지식 이름: "고객 피드백 모음"
지식 설명: "고객 상담 중 수집된 피드백"
텍스트 콘텐츠: [피드백 내용]
```

**예시 3: 기존 문서에 내용 누적**

```
지식 선택: "FAQ 문서"
파일명: "자주묻는질문.txt"
텍스트 콘텐츠: [새로운 Q&A 내용]
기존 문서에 추가: 켬
```

### 인덱싱 상태 확인 도구

지식에 추가한 문서의 처리 상태를 확인합니다. 문서가 검색 가능한 상태가 되었는지 모니터링할 때 사용합니다.

#### 설정 항목

아래 항목 중 최소 하나를 입력해야 합니다.

| 항목        | 설명                       |
| --------- | ------------------------ |
| **지식 ID** | 상태를 확인할 지식의 ID           |
| **배치 ID** | 지식 추가 시 반환된 배치 ID        |
| **문서 ID** | 특정 문서의 ID (여러 개는 쉼표로 구분) |

#### 전체 상태 해석

여러 문서를 동시에 처리할 때의 전체 상태입니다.

| 상태             | 의미             |
| -------------- | -------------- |
| **completed**  | 모든 문서 처리 완료    |
| **processing** | 처리 중인 문서 존재    |
| **partial**    | 일부 완료, 일부 처리 중 |
| **error**      | 모든 문서 처리 실패    |

### 실전 활용 가이드

#### 두 도구 함께 사용하기

지식 추가 후 인덱싱 완료까지 확인하는 일반적인 패턴입니다.

```
1. [지식 추가 도구] 문서를 지식에 추가
   → dataset_id, document_ids, batch 값 반환

2. [인덱싱 상태 확인 도구] 반환된 값으로 상태 확인
   → status가 "completed"인지 확인

3. 완료될 때까지 주기적으로 상태 확인
```

#### 추천 사용 패턴

**패턴 1: 대화 내용 자동 아카이브**

* 대화 종료 시점에 중요 내용을 요약하여 지식에 저장
* 같은 주제의 지식에 계속 누적 (기존 문서에 추가: 켬)

**패턴 2: 파일 업로드 후 즉시 활용**

* 사용자가 파일 업로드 → 지식 추가 도구로 저장
* 인덱싱 상태 확인 → 완료 후 해당 지식을 검색에 활용

**패턴 3: 정보 수집 및 축적**

* 외부 API나 웹에서 가져온 정보를 텍스트로 저장
* 날짜별 파일명으로 정리 (예: "2024-12-28-뉴스요약.txt")

### FAQ

#### Q: 문서 추가 후 바로 검색이 안 돼요

**A**: 문서가 인덱싱되는 데 시간이 필요합니다. 인덱싱 상태 확인 도구로 상태가 "완료"인지 확인하세요. 문서 크기에 따라 수 초에서 수 분이 걸릴 수 있습니다.

#### Q: "오류" 상태가 표시돼요

**A**: 문서 형식이 지원되지 않거나 파일이 손상되었을 수 있습니다. 다른 형식으로 변환하거나 파일을 다시 업로드해 보세요.

#### Q: 같은 파일을 여러 번 추가하면 어떻게 되나요?

**A**: "기존 문서에 추가" 설정에 따라 다릅니다.

* **끔**: 같은 이름의 새 문서가 별도로 생성됩니다.
* **켬**: 기존 문서의 끝에 새 내용이 추가됩니다.

***

### 팁

* 지식 설명을 상세히 작성하면 AI가 더 정확하게 정보를 분류할 수 있습니다.
* 파일명에 날짜를 포함하면 나중에 관리하기 편리합니다.
* 정기적으로 추가되는 정보는 "기존 문서에 추가" 기능을 활용하세요.
* 인덱싱이 완료되기 전에는 해당 문서를 검색할 수 없습니다.
