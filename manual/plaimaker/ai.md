# AI 어시스턴트 활용하기

AI 어시스턴트는 자연어로 요청을 전달하면 앱 생성, 수정, 구조 설계 등 다양한 작업을 도와주는 기능입니다. 사용자는 필요한 기능이나 원하는 결과를 설명하는 것만으로도 앱 구성을 빠르게 만들거나 기존 내용을 효율적으로 개선할 수 있습니다.

{% hint style="info" %}
#### AI 어시스턴트는 어떤 기능을 지원하나요?

AI 어시스턴트는 기본적으로 미소 앱에서 사용자가 직접 편집할 수 있는 모든 기능 범위 내에서 동작합니다.

**생성하기**\
빈 영역에서 새로운 앱 제작을 요청할 수 있습니다. 필요한 기능이나 구조를 설명하면 AI 어시스턴트가 이를 기반으로 앱 구성을 생성합니다.

**수정하기**\
이미 생성된 앱의 내용이나 구조를 변경하거나 개선할 수 있습니다. 원하는 변경 사항을 설명하면 해당 내용을 반영하여 수정 작업을 수행합니다.

**질문하기(적용 예정)**\
편집 중인 미소 앱과 관련된 질문을 할 수 있는 기능입니다. 현재는 개발 중이며, 추후 업데이트를 통해 적용될 예정입니다.
{% endhint %}

## 1. AI 어시스턴트 활성화하기

앱 편집 화면의 우측 상단에 있는 **AI 어시스턴트** 버튼을 클릭하면 채팅 창이 활성화됩니다.

<figure><img src="../../.gitbook/assets/image (603).png" alt="" width="563"><figcaption></figcaption></figure>

{% hint style="warning" %}
#### AI 에이전트 버튼이 안보여요

AI 에이전트 기능은 실험실 기능으로 제공되며, 관리자 페이지에서 기능을 **ON**으로 설정한 경우에만 활성화됩니다. 사용이 필요하신 경우 MISO 담당 관리자에게 문의하세요.
{% endhint %}

## 2. 앱 유형 별 활용 가이드

### 워크플로우

#### 1) 화면 가이드

<figure><img src="../../.gitbook/assets/image (606).png" alt=""><figcaption></figcaption></figure>

* **하단 채팅 창에 지시사항을 입력하여 앱 편집을 요청할 수 있습니다.**
* **새 대화 : 현재 대화를 종료 후 새로운 대화에 진입합니다. 이전 대화는 복원되지 않습니다.**
*   **컨텍스트 추가 : 대상 노드를 지정하여 보다 정확한 지시를 할 수 있는 기능입니다.**

    * 사용자가 클릭하여 상세 창이 활성화된 노드는 자동으로 추가됩니다.
    * 컨텍스트 추가 버튼 클릭 - 추가할 노드 클릭하여 여러 개의 노드를 지정할 수 있습니다.

#### 2) 앱 편집하기

* 작업 지시를 입력하면 AI 어시스턴트가 내용을 분석하여 작업을 계획합니다.
* 만약 사용자의 요청이 모호하거나 작업을 진행하기에 필요한 정보가 부족한 경우, 보다 구체적인 내용을 확인하기 위한 설문 형태의 질문이 표시되어 추가 정보를 입력할 수 있도록 안내합니다.

<figure><img src="../../.gitbook/assets/image (605).png" alt=""><figcaption></figcaption></figure>

* 사용자의 구체적인 계획이 파악되었을 경우, 아래와 같은 작업 계획을 먼저 출력합니다. 이후 사용자가 해당 계획을 승인하면 작업을 시작합니다.

<figure><img src="../../.gitbook/assets/Clipboard-20260312-015332-305.gif" alt=""><figcaption></figcaption></figure>

* AI 어시스턴트가 앱 편집을 완료하면 수정된 내용을 요약하여 보여줍니다. 하단의 **취소** 또는 **승인** 버튼을 통해 해당 변경 사항을 반영할지 여부를 결정할 수 있습니다.

<figure><img src="../../.gitbook/assets/image (597).png" alt="" width="375"><figcaption></figcaption></figure>

{% hint style="danger" %}
AI 어시스턴트가 변경한 내용을 확인한 뒤 **취소** 또는 **승인**을 선택할 수 있습니다.&#x20;

단, 한 번 **승인**한 이후에는 이전 상태로 되돌릴 수 없으므로 신중하게 선택해야 합니다.&#x20;

안정적인 운영을 위해 AI 어시스턴트를 사용하기 전에는 기존 워크플로우를 먼저 발행해두는 것을 권장합니다.
{% endhint %}

### 에이전트

#### 1) 화면 가이드

* 우측 채팅 창에서 워크플로우 AI 어시스턴트와 동일한 방식으로 요청할 수 있습니다.&#x20;
  * 단, AI 어시스턴트에는 워크플로우와 같은 **컨텍스트 추가** UI가 별도로 제공되지 않으며, 모든 요청은 채팅창을 통해서만 입력합니다.
* AI 어시스턴트에게 편집을 요청하면 작업 결과에 따라 변경 사항이 가운데 영역에 표시됩니다.
  * 체크박스를 클릭하여 변경사항 적용 여부를 선택한 뒤, 우측 하단의 **선택 적용** 버튼을 클릭하면 해당 내용이 반영됩니다.

<figure><img src="../../.gitbook/assets/image (598).png" alt=""><figcaption></figcaption></figure>

#### 2) 앱 편집하기

사용자는 AI 어시스턴트에게 **원하는 앱의 목적, 입력 방식, 응답 조건**을 자연어로 구체적으로 설명하여 편집을 요청할 수 있습니다.

예를 들어 다음과 같이 요청할 수 있습니다.

* **“주간 식단표를 기반으로 오늘 메뉴를 알려주는 앱 만들어줘”**
* **“파일 입력 기능을 활성화해서 사용자가 업로드한 파일을 기반으로도 대답할 수 있게 해줘”**
* **“입력변수에 오늘의 기분을 입력받고 사용자의 기분에 따라 응답 톤을 다르게 해줘”**

이처럼 **기능, 입력값, 응답 방식까지 함께 지시**할 수 있습니다.

요청 내용이 **구체적일수록 AI 어시스턴트가 의도를 더 정확하게 이해**하여, 원하는 **구조와 설정을 빠르게 반영**할 수 있습니다.
