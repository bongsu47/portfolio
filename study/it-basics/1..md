# 1. 서비스의 구조

## 서비스는 어떻게 만들어지나요?

매일 아침 스마트폰을 열어 날씨를 확인하고, 카페 앱으로 커피를 주문하고, 메신저로 동료에게 메시지를 보냅니다.\
이 모든 것이 IT 서비스입니다.

그런데 화면에서 버튼 하나를 누르는 그 짧은 순간, 뒤에서는 어떤 일이 벌어지고 있을까요?\
이 글은 그 질문에서 시작합니다.

***

### 카페 한 잔에 담긴 IT의 구조

카페에서 음료를 주문하는 과정을 떠올려 보겠습니다.

1. 손님이 **메뉴판**을 보고 "아이스 라떼 한 잔"을 주문합니다
2. 주문이 **주문 창구**를 통해 주방으로 전달됩니다
3. **주방**에서 **재료 창고**의 원두와 우유를 꺼내 음료를 만듭니다
4. 완성된 음료가 손님에게 전달됩니다

놀랍게도, IT 서비스도 정확히 같은 구조로 동작합니다.

<figure><img src="../../.gitbook/assets/image (610).png" alt="" width="375"><figcaption></figcaption></figure>

| 카페                    | IT 서비스         | 역할               |
| --------------------- | -------------- | ---------------- |
| **손님** (메뉴판을 보고 주문)   | **화면** (클라이언트) | 사용자가 보고 조작하는 부분  |
| **주문 창구** (주문을 전달)    | **네트워크**       | 화면과 서버를 연결하는 통로  |
| **주방** (음료를 만드는 곳)    | **서버**         | 요청을 받아 처리하는 곳    |
| **재료 창고** (원두, 우유 보관) | **데이터베이스**     | 데이터를 저장하고 관리하는 곳 |

{% hint style="info" %}
**클라이언트**는 "요청하는 쪽"이라는 뜻입니다. 카페에서 손님(client)이 음료를 요청하듯, IT에서도 화면이 서버에게 데이터를 요청하기 때문에 클라이언트라고 부릅니다.
{% endhint %}

이 네 가지가 IT 서비스의 전부입니다.\
아무리 복잡한 서비스라도, 결국은 이 구조 안에서 동작합니다.

***

### 실제 서비스에서는 어떻게 작동하나요?

음악 스트리밍 서비스에서 노래를 검색하는 상황을 예로 들어 보겠습니다.

<table><thead><tr><th width="90.91015625">단계</th><th>일어나는 일</th><th>카페 비유</th></tr></thead><tbody><tr><td>1</td><td>사용자가 검색창에 "아이유"를 입력하고 검색 버튼을 누릅니다</td><td>손님이 메뉴판에서 음료를 고릅니다</td></tr><tr><td>2</td><td>검색 요청이 네트워크를 통해 서버로 전달됩니다</td><td>주문이 주문 창구를 통해 주방으로 갑니다</td></tr><tr><td>3</td><td>서버가 데이터베이스에서 "아이유"와 관련된 곡 목록을 찾아옵니다</td><td>주방에서 재료 창고의 재료를 꺼냅니다</td></tr><tr><td>4</td><td>검색 결과가 다시 네트워크를 통해 화면에 표시됩니다</td><td>완성된 음료가 손님에게 전달됩니다</td></tr></tbody></table>

로그인, 메시지 전송, 파일 업로드 - 어떤 기능이든 이 흐름을 따릅니다.\
**화면 → 네트워크 → 서버 → 데이터베이스**, 그리고 다시 역순으로 돌아오는 것입니다.

***

### 각 영역은 누가 만드나요?

| 영역         | 담당            | 사용 기술                 |
| ---------- | ------------- | --------------------- |
| 화면 (클라이언트) | 프론트엔드 개발자     | HTML, CSS, JavaScript |
| 서버         | 백엔드 개발자       | Python, Java, Node.js |
| 데이터베이스     | 백엔드 개발자 / DBA | MySQL, PostgreSQL     |

{% hint style="info" %}
**프론트엔드**는 "앞쪽", **백엔드**는 "뒷쪽"이라는 뜻입니다. 사용자에게 보이는 화면을 만드는 사람이 프론트엔드 개발자, 보이지 않는 서버를 만드는 사람이 백엔드 개발자입니다. **DBA**(Database Administrator)는 데이터베이스를 전문적으로 관리하는 사람입니다.
{% endhint %}

이 모든 영역은 **운영체제(OS)**&#xB77C;는 기반 위에서 동작합니다. Windows, macOS, Linux 등이 하드웨어를 관리하고, 그 위에서 프로그램이 실행됩니다.

***

### 한 줄 정리

> IT 서비스는 **화면 → 네트워크 → 서버 → 데이터베이스**의 흐름으로 동작합니다. 카페에서 손님이 주문하고, 주방에서 재료를 꺼내 음료를 만들어 전달하는 것과 같은 구조입니다.

이제 이 구조를 하나씩 들여다보겠습니다.\
가장 먼저, 사용자가 직접 만나는 **화면**부터 시작합니다.
