# 2. 화면의 세계

## 사용자가 만나는 화면

1장에서 카페에 비유했던 구조를 기억하시나요?\
손님이 메뉴판을 보고 음료를 고르는 그 순간이 IT 서비스에서는 사용자가 **화면**을 보고 있는 순간입니다.

스마트폰의 앱을 열거나, 컴퓨터에서 웹사이트에 접속할 때 가장 먼저 눈에 들어오는 것이 바로 이 화면입니다.\
IT에서는 이것을 **클라이언트**라고 부릅니다.

***

### 클라이언트란 무엇인가요?

클라이언트는 사용자가 직접 보고 조작하는 프로그램입니다.

예를 들어,

* 스마트폰에서 사용하는 앱
* 컴퓨터의 웹 브라우저에서 여는 웹사이트
* 태블릿에서 실행하는 앱

이 모두가 클라이언트입니다. 사용자의 행동을 서버에 전달하고, 서버에서 돌아온 결과를 화면에 보여주는 역할을 합니다.

클라이언트는 여러 이름으로 불립니다.

| 호칭                    | 의미                       |
| --------------------- | ------------------------ |
| **클라이언트**             | 정식 명칭                    |
| **프론트엔드 (front-end)** | "앞쪽"이라는 뜻으로, 사용자에게 보이는 쪽 |
| **프론트**               | 프론트엔드의 줄임말               |

***

### 화면은 무엇으로 만드나요?

클라이언트는 세 가지 핵심 기술로 만들어집니다.

<figure><img src="../../.gitbook/assets/image (30).png" alt="" width="375"><figcaption></figcaption></figure>

| 기술             | 역할           | 비유           |
| -------------- | ------------ | ------------ |
| **HTML**       | 화면의 뼈대 (구조)  | 건물의 골조       |
| **CSS**        | 화면의 꾸밈 (디자인) | 건물의 인테리어     |
| **JavaScript** | 화면의 동작 (기능)  | 건물의 전기·배관 설비 |

이 세 가지에 이미지, 폰트 같은 디자인 요소가 더해져 우리가 보는 화면이 완성됩니다.

{% hint style="info" %}
HTML, CSS, JavaScript는 모두 프로그래밍 기술의 이름입니다. 지금은 이름을 외울 필요 없이, "화면은 세 가지 기술로 만들어진다"는 정도만 기억하면 충분합니다. 7장에서 프로그래밍 언어를 더 자세히 다룹니다.
{% endhint %}

***

### 앱과 웹, 무엇이 다른가요?

화면이 사용자에게 전달되는 방식에는 크게 **앱**과 **웹** 두 가지가 있습니다.

#### 앱 (App)

* 기기에 **설치**해서 사용합니다
* App Store, Google Play 등을 통해 배포됩니다
* 새 버전이 나오면 사용자가 **직접 업데이트**해야 합니다

{% hint style="info" %}
**배포**란 만들어진 프로그램을 사용자가 쓸 수 있도록 공개하는 것을 말합니다. 앱은 스토어에 올리는 것이 배포이고, 웹은 서버에 올려서 접속할 수 있게 하는 것이 배포입니다.
{% endhint %}

#### 웹 (Web)

* **브라우저**에서 주소를 입력하여 접속합니다
* 설치 없이 바로 사용할 수 있습니다
* 서비스 제공자가 업데이트하면 사용자는 **자동으로 최신 버전**을 사용하게 됩니다

| 구분    | 앱                   | 웹              |
| ----- | ------------------- | -------------- |
| 설치    | 필요                  | 불필요            |
| 접근 방법 | 스토어에서 다운로드          | 브라우저에서 주소 입력   |
| 업데이트  | 사용자가 직접             | 자동 반영          |
| 사용 환경 | 특정 기기(iOS, Android) | 브라우저만 있으면 어디서든 |

***

### 브라우저란?

**브라우저(Browser)**&#xB294; 웹 페이지를 열어주는 프로그램입니다.

| 브라우저   | 만든 회사     |
| ------ | --------- |
| Chrome | Google    |
| Edge   | Microsoft |
| Safari | Apple     |
| Whale  | Naver     |

운영체제마다 기본 브라우저가 다르지만, 어떤 운영체제를 쓰든 브라우저만 있으면 동일한 웹 서비스에 접속할 수 있습니다. 이것이 웹의 큰 장점입니다.

브라우저에는 **확장 프로그램**을 설치하여 기능을 추가할 수도 있습니다. 광고 차단, 번역 도우미 등 다양한 확장 프로그램이 제공됩니다.

{% hint style="warning" %}
확장 프로그램은 외부에서 가져오는 것이므로, 신뢰할 수 있는 제공자인지 확인하는 것이 좋습니다.
{% endhint %}

***

### 한 줄 정리

> 클라이언트(화면)는 사용자가 직접 보고 조작하는 부분입니다. 기기에 설치하는 "앱"과 브라우저로 접속하는 "웹" 두 가지 방식이 있으며, 어느 쪽이든 서버와 데이터를 주고받는 역할은 동일합니다.

화면에서 버튼 하나를 누르는 순간, 눈에 보이지 않는 일들이 시작됩니다.\
다음 장에서는 그 **요청과 응답**의 세계를 들여다봅니다.
