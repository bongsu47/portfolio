# 6. 데이터 관리

## 데이터는 어디에 저장되나요?

음악 스트리밍 서비스를 생각해 봅시다.\
수천만 곡의 음악 정보, 수억 명의 사용자 정보, 그리고 각 사용자가 만든 플레이리스트 등 서버가 이 모든 데이터를 처리한 뒤, 결과를 어딘가에 보관해야 합니다.

그 "어딘가"가 바로 **데이터베이스**입니다.

***

### 왜 데이터베이스가 필요한가요?

만약 모든 데이터를 일반 파일로 관리한다면 어떻게 될까요?

음악 스트리밍 서비스의 데이터를 텍스트 파일에 저장한다고 상상해 봅시다.

```
아이유, 2008년 데뷔, 팝/발라드, LILAC, 2021년... / BTS, 2013년 데뷔, 팝/힙합, BE, 2020년...
```

#### 문제 1: 데이터가 뒤섞인다

하나의 파일에 아티스트 정보, 앨범 정보, 재생 기록이 모두 섞여 들어갑니다.\
"이 아티스트의 앨범 목록만 보여줘"라는 요청을 처리하기 매우 어렵습니다.

#### 문제 2: 데이터가 많아지면 느려진다

파일이 적을 때는 검색이 빠릅니다.\
하지만 수천만 곡의 데이터가 쌓이면, 원하는 한 곡을 찾는 데 매우 오랜 시간이 걸립니다.

이런 문제를 해결하기 위해 등장한 것이 데이터베이스입니다.

***

### 관계형 데이터베이스란?

**관계형 데이터베이스(RDB)**&#xB294; 데이터를 **테이블(Table)** 형태로 정리하여 저장합니다.\
스프레드시트(엑셀)의 시트와 비슷하다고 생각하면 됩니다.

{% hint style="info" %}
**테이블**은 엑셀의 시트 한 장과 같습니다. 가로 줄(행)은 하나의 데이터 항목이고, 세로 줄(열)은 그 항목의 속성(이름, 나이 등)입니다. **관계형**이라는 이름은 여러 테이블이 서로 연결(관계)되어 있기 때문에 붙은 것입니다.
{% endhint %}

핵심은 데이터의 종류별로 **테이블을 나누고**, 테이블 사이를 **연결(관계)**&#xD558;는 것입니다.

***

### 테이블은 어떻게 구성되나요?

음악 스트리밍 서비스를 예로 들어 보겠습니다.

#### 아티스트 테이블

| 번호 | 이름  | 데뷔년도 | 장르    |
| -- | --- | ---- | ----- |
| 1  | 아이유 | 2008 | 팝/발라드 |
| 2  | BTS | 2013 | 팝/힙합  |
| 3  | 뉴진스 | 2022 | 팝     |

#### 앨범 테이블

| 번호 | 앨범명       | 아티스트    | 발매년도 |
| -- | --------- | ------- | ---- |
| 1  | LILAC     | 1 (아이유) | 2021 |
| 2  | Love poem | 1 (아이유) | 2019 |
| 3  | BE        | 2 (BTS) | 2020 |

이렇게 테이블을 나누면, "아이유의 앨범 목록"을 찾을 때 앨범 테이블에서 아티스트 번호가 1인 행만 검색하면 됩니다. 데이터가 아무리 많아도 빠르고 정확하게 찾을 수 있습니다.

***

### 관계형 데이터베이스의 핵심 개념

{% endhint %}

### 데이터베이스 안의 데이터는 JSON과 어떤 관계인가요?

4장에서 배운 JSON과 데이터베이스는 서로 다른 역할을 합니다.

| 구분     | 데이터베이스 (테이블)    | JSON             |
| ------ | --------------- | ---------------- |
| **역할** | 데이터를 **저장**하는 곳 | 데이터를 **전송**하는 형식 |
| **비유** | 재료 창고의 선반       | 재료를 담아 나르는 상자    |

예를 들어, 사용자가 앨범 목록을 요청하면:

1. 서버가 데이터베이스의 **테이블**에서 데이터를 꺼냅니다
2. 꺼낸 데이터를 **JSON** 형식으로 포장합니다
3. 네트워크를 통해 화면에 전달합니다

```json
{
  "아티스트": "아이유",
  "앨범목록": [
    {"앨범명": "LILAC", "발매년도": 2021},
    {"앨범명": "Love poem", "발매년도": 2019}
  ]
}
```

데이터베이스에 저장된 테이블의 내용이, JSON이라는 상자에 담겨 화면까지 전달되는 것입니다.

***

### 데이터베이스의 종류

| 종류               | 대표 기술             | 특징                     |
| ---------------- | ----------------- | ---------------------- |
| **관계형 (RDB)**    | MySQL, PostgreSQL | 테이블 형태, 정형 데이터에 적합     |
| **비관계형 (NoSQL)** | MongoDB, Redis    | 유연한 형태, 대량·비정형 데이터에 적합 |

{% hint style="info" %}
**정형 데이터**는 엑셀 표처럼 정해진 형식에 딱 맞는 데이터(이름, 나이, 주소 등)이고, **비정형 데이터**는 형식이 자유로운 데이터(채팅 내용, 이미지, 로그 기록 등)입니다. 관계형 DB는 정형 데이터에 강하고, NoSQL은 비정형 데이터에 유연하게 대응합니다.
{% endhint %}

대부분의 서비스에서는 관계형 데이터베이스를 기본으로 사용하고, 필요에 따라 비관계형 데이터베이스를 함께 사용합니다.

***

### 한 줄 정리

> 데이터베이스는 데이터를 테이블 형태로 체계적으로 저장하고 관리하는 시스템입니다. 관계형 데이터베이스(RDB)는 테이블 간의 관계를 이용하여, 수천만 건의 데이터에서도 원하는 정보를 빠르고 정확하게 찾아냅니다.

여기까지 IT 서비스가 동작하는 전체 구조를 살펴보았습니다.\
화면, 네트워크, 서버, 데이터베이스까지 이 모든 것은 **프로그램**입니다.\
