# 7. 소프트웨어의 기반

## 프로그램은 어떻게 만들어지나요?

지금까지 살펴본 화면, 서버, 데이터베이스는 모두 **프로그램**입니다.\
그런데 프로그램은 대체 어떻게 만들까요?

요리사가 레시피를 보고 요리를 만들듯, 개발자는 **프로그래밍 언어**로 레시피(코드)를 작성하고 컴퓨터가 이를 실행합니다.\
그리고 이 모든 프로그램은 **운영체제**라는 기반 위에서 동작합니다.

***

### 프로그래밍 언어란?

프로그래밍 언어는 사람이 컴퓨터에게 명령을 내리기 위해 사용하는 언어입니다.

요리에 비유하면 이렇습니다.

| 요리                   | 프로그래밍                   |
| -------------------- | ----------------------- |
| **레시피** (재료와 조리 순서)  | **코드** (명령을 순서대로 작성한 것) |
| **요리사** (레시피를 읽고 요리) | **컴퓨터** (코드를 읽고 실행)     |
| **완성된 요리**           | **프로그램** (실행 가능한 소프트웨어) |

요리사가 레시피의 각 단계를 순서대로 따라가듯, 컴퓨터도 코드에 적힌 명령을 위에서부터 하나씩 실행합니다.

***

### 코드는 어떻게 프로그램이 되나요?

컴퓨터는 사람의 말을 직접 이해하지 못합니다.\
그래서 프로그래밍 언어로 작성된 코드를 컴퓨터가 이해할 수 있는 형태로 바꿔주는 과정이 필요합니다.

1. 개발자가 프로그래밍 언어로 **코드(레시피)**&#xB97C; 작성합니다
2. **컴파일러** 또는 **인터프리터**가 코드를 변환합니다
3. 변환된 결과가 **프로그램**이 됩니다
4. 컴퓨터가 이 프로그램을 실행합니다

{% hint style="info" %}
**컴파일러**는 레시피 전체를 한 번에 변환하는 방식이고, **인터프리터**는 한 줄씩 읽으며 바로 실행하는 방식입니다.
{% endhint %}

***

### 자주 만나게 될 두 가지 언어

프로그래밍 언어는 수십 가지가 있지만, IT 서비스를 이해하는 데 특히 자주 등장하는 두 가지를 소개합니다.

#### Python (파이썬)

| 항목          | 설명                                  |
| ----------- | ----------------------------------- |
| **특징**      | 문법이 간결하고 읽기 쉬움. "프로그래밍의 입문 언어"로 유명  |
| **주요 용도**   | 서버(백엔드), 데이터 분석, AI·머신러닝            |
| **어디서 만나나** | AI 서비스의 서버, 데이터 처리 자동화, 업무 자동화 스크립트 |

Python은 마치 "쉬운 레시피"와 같습니다. 복잡한 기법 없이도 요리(프로그램)를 만들 수 있어서, 비전공자도 배우기 쉽습니다. 최근 AI 분야가 급성장하면서 가장 인기 있는 언어가 되었습니다.

#### JavaScript (자바스크립트)

| 항목          | 설명                                      |
| ----------- | --------------------------------------- |
| **특징**      | 웹 브라우저에서 바로 실행되는 유일한 언어                 |
| **주요 용도**   | 웹 화면(프론트엔드), 서버(Node.js)                |
| **어디서 만나나** | 웹사이트의 모든 동적 기능 (버튼 클릭, 애니메이션, 실시간 업데이트) |

2장에서 화면을 만드는 세 가지 기술(HTML, CSS, JavaScript)을 소개했습니다. 그중 JavaScript는 화면에 **동작**을 부여하는 역할을 합니다. 버튼을 누르면 메뉴가 열리고, 스크롤하면 새 내용이 로딩되는 것들이 JavaScript입니다.

{% hint style="info" %}
**Java와 JavaScript는 전혀 다른 언어입니다.** 이름이 비슷하지만 문법도, 용도도 다릅니다. "자바"와 "자바스크립트"를 혼동하지 않도록 주의하세요.
{% endhint %}

#### 그 밖의 언어들

| 언어         | 주요 용도             |
| ---------- | ----------------- |
| **Java**   | 대규모 서버, Android 앱 |
| **Swift**  | iOS/macOS 앱       |
| **Kotlin** | Android 앱         |
| **C#**     | Windows 프로그램, 게임  |

하나의 언어로 모든 것을 만들 수 있는 것은 아닙니다. 한식 레시피와 양식 레시피가 다른 재료와 도구를 쓰듯, 각 분야에 적합한 언어가 따로 있습니다.

***

### 운영체제 : 프로그램이 동작하는 무대

프로그램이 실행되려면 컴퓨터의 부품(CPU, 메모리, 저장장치 등)을 사용해야 합니다.\
**운영체제(OS)**&#xB294; 이 부품들을 관리하고, 프로그램이 동작할 수 있는 환경을 제공하는 기본 소프트웨어입니다.

요리에 비유하면, 운영체제는 **주방 시설 전체**에 해당합니다.

| 주방 시설  | 컴퓨터 부품             | 역할                     |
| ------ | ------------------ | ---------------------- |
| 가스레인지  | **CPU** (프로세서)     | 연산과 처리를 담당하는 "두뇌"      |
| 조리대    | **RAM** (메모리)      | 작업 중인 데이터를 임시로 올려놓는 공간 |
| 냉장고·창고 | **HDD/SSD** (저장장치) | 데이터를 영구적으로 보관          |

운영체제는 이 부품들이 서로 잘 협력하도록 조율합니다.

#### 운영체제의 종류

| 운영체제        | 만든 곳      | 사용 기기        |
| ----------- | --------- | ------------ |
| **Windows** | Microsoft | PC, 노트북      |
| **macOS**   | Apple     | Mac          |
| **iOS**     | Apple     | iPhone, iPad |
| **Android** | Google    | 스마트폰, 태블릿    |
| **Linux**   | 오픈소스 커뮤니티 | 서버, 개발 환경    |

{% hint style="info" %}
**오픈소스**란 프로그램의 설계도(소스 코드)가 누구에게나 공개되어 있어, 전 세계 개발자들이 함께 만들고 개선하는 방식입니다. Linux는 이런 방식으로 만들어진 대표적인 운영체제이며, 대부분의 서버에서 사용됩니다.
{% endhint %}

#### 프로그램과 운영체제의 관계

프로그램은 **특정 운영체제 위에서** 동작합니다.

* Windows용 프로그램은 macOS에서 실행할 수 없습니다
* iPhone 앱(iOS)은 Android 폰에서 실행할 수 없습니다

이 때문에 같은 서비스라도 iOS용, Android용, 웹용을 따로 만드는 경우가 많습니다.

반면, 하나의 운영체제 위에는 여러 프로그램을 설치할 수 있습니다. 스마트폰에 카메라 앱, 메신저, 음악 앱을 함께 설치하는 것이 대표적인 예입니다.

***

### 코드와 보안

프로그램은 수많은 코드로 이루어져 있습니다. 이 코드에 **취약점**이 숨어 있을 수 있습니다.

{% hint style="info" %}
**취약점**이란 프로그램에 있는 보안상의 허점입니다. 마치 건물의 잠기지 않은 뒷문과 같아서, 악의적인 사용자가 이를 이용해 데이터를 훔치거나 시스템을 망가뜨릴 수 있습니다.
{% endhint %}

예를 들어,

* 파일 업로드 기능이 있는 서비스에서, 악성 파일이 업로드될 수 있습니다
* 이를 방지하기 위해 업로드되는 파일의 안전성을 검사하는 코드를 작성합니다

운영체제 수준에서도 보안 관리가 이루어집니다. 신뢰할 수 없는 프로그램의 설치를 제한하거나, 악성 코드를 탐지하는 기능이 운영체제에 포함되어 있습니다.

***

### 한 줄 정리

> 프로그래밍 언어는 개발자가 컴퓨터에게 명령을 전달하기 위한 도구이며, 운영체제는 프로그램이 동작할 수 있는 환경을 제공합니다. 웹 화면은 주로 **JavaScript**로, AI 서비스의 서버는 주로 **Python**으로 만들어집니다.

여기까지 IT 서비스를 만드는 기술적 토대를 살펴보았습니다.\
마지막 장에서는 우리가 매일 사용하는 **검색**이 어떻게 진화하고 있는지 알아봅니다.
