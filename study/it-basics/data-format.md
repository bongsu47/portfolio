# 4. 데이터의 생김새

## 데이터의 생김새

택배를 보낼 때 물건을 그냥 보내지 않습니다. 상자에 담고, 뽁뽁이로 감싸고, 송장을 붙입니다.\
데이터도 마찬가지입니다. 화면과 서버 사이를 오가는 데이터에는 **형태**가 있고, 정해진 **포맷**으로 포장되어 전달됩니다.

이번 장에서는 데이터가 어떤 종류로 나뉘는지, 그리고 어떤 포맷으로 주고받는지를 살펴봅니다.

***

### 데이터에도 종류가 있나요?

카페 주문서를 생각해 봅시다. 주문서에는 다양한 종류의 정보가 적힙니다.

| 주문서 항목   | 예시       | 데이터 종류             |
| -------- | -------- | ------------------ |
| 메뉴 이름    | "아이스 라떼" | **텍스트 (string)**   |
| 수량       | 2        | **숫자 (number)**    |
| 얼음 많이 여부 | 예 / 아니오  | **참/거짓 (boolean)** |

이처럼 데이터는 종류에 따라 다르게 다룹니다. IT에서는 이것을 **데이터 타입(자료형)**&#xC774;라고 부릅니다.

{% hint style="info" %}
**데이터 타입**은 "이 칸에는 어떤 종류의 정보가 들어가나요?"를 정해놓은 것입니다. 입력 양식에서 "이름" 칸에는 글자를, "나이" 칸에는 숫자를 쓰도록 정해져 있는 것과 같습니다.
{% endhint %}

#### 기본 타입

| 타입          | 의미       | 예시                     |
| ----------- | -------- | ---------------------- |
| **string**  | 텍스트 (글자) | "아이스 라떼", "홍길동", "서울시" |
| **number**  | 숫자       | 2, 4500, 3.14          |
| **boolean** | 참 또는 거짓  | true(예), false(아니오)    |

#### 묶음 타입

데이터가 하나가 아니라 **여러 개**일 때는 어떻게 할까요?

**배열 (array)**&#xC740; 같은 종류의 데이터를 **순서대로 나열**한 것입니다.

쇼핑 목록을 떠올리면 됩니다. "우유, 빵, 계란"처럼 항목을 나열하는 것입니다.

| 타입                | 의미      | 예시                       |
| ----------------- | ------- | ------------------------ |
| **array(string)** | 텍스트의 목록 | \["라떼", "아메리카노", "카푸치노"] |
| **array(number)** | 숫자의 목록  | \[4500, 3000, 5000]      |

**객체 (object)**&#xB294; 서로 다른 종류의 데이터를 **이름표**와 함께 묶은 것입니다.

신분증을 떠올리면 됩니다. 이름, 생년월일, 주소가 각각의 칸에 적혀 있듯이, 여러 정보를 하나로 묶을 수 있습니다.

| 이름표   | 값     |
| ----- | ----- |
| 이름    | "홍길동" |
| 나이    | 25    |
| 회원 여부 | true  |

**array(object)**&#xB294; 객체가 여러 개 모인 것입니다. 회원 명단처럼, 각 사람의 정보 묶음이 목록으로 나열된 형태입니다.

{% code expandable="true" %}
```
{
  "이름": "홍길동",
  "나이": 25,
  "회원여부": true
}
```
{% endcode %}

{% hint style="info" %}
정리하면, **string·number·boolean**은 값 하나를 담는 기본 타입이고, **array**는 여러 값을 순서대로 나열한 것, **object**는 여러 값을 이름표와 함께 묶은 것입니다. 이 다섯 가지만 알면 대부분의 데이터 구조를 이해할 수 있습니다.
{% endhint %}

***

### JSON : 가장 널리 쓰이는 데이터 포맷

데이터 타입을 알았으니, 이제 데이터를 **어떤 형식으로** 주고받는지 살펴보겠습니다.

**JSON(JavaScript Object Notation)**&#xC740; 현재 가장 널리 쓰이는 데이터 포맷입니다.\
사람이 읽기 쉽고, 컴퓨터가 처리하기도 쉬워서 대부분의 API가 JSON을 사용합니다.

카페 주문을 JSON으로 표현하면 이렇습니다.

```json
{
  "메뉴": "아이스 라떼",
  "수량": 2,
  "얼음많이": true,
  "옵션": ["샷 추가", "덜 달게"]
}
```

규칙은 간단합니다.

| 기호        | 의미              | 예시                |
| --------- | --------------- | ----------------- |
| `{ }`     | 객체 (이름표가 붙은 묶음) | `{"이름": "홍길동"}`   |
| `[ ]`     | 배열 (순서가 있는 목록)  | `["라떼", "아메리카노"]` |
| `"이름": 값` | 이름표와 값의 쌍       | `"수량": 2`         |

{% hint style="info" %}
JSON에서 텍스트는 반드시 큰따옴표(`" "`)로 감싸야 합니다. 숫자와 boolean(true/false)은 따옴표 없이 씁니다.
{% endhint %}

JSON은 **중첩**도 가능합니다. 마치 서랍장 안에 작은 서랍이 또 있는 것처럼, 객체 안에 배열이, 배열 안에 객체가 들어갈 수 있습니다.

```json
{
  "주문자": "홍길동",
  "주문목록": [
    {"메뉴": "아이스 라떼", "수량": 1},
    {"메뉴": "아메리카노", "수량": 2}
  ]
}
```

특정 값을 가리킬 때는 **점(.)과 대괄호**를 사용합니다.

| 표현           | 의미          | 결과       |
| ------------ | ----------- | -------- |
| `주문자`        | 주문자의 이름     | "홍길동"    |
| `주문목록[0].메뉴` | 첫 번째 주문의 메뉴 | "아이스 라떼" |
| `주문목록[1].수량` | 두 번째 주문의 수량 | 2        |

{% hint style="info" %}
배열의 순서는 0부터 시작합니다. 첫 번째 항목이 `[0]`, 두 번째가 `[1]`입니다.
{% endhint %}

***

### XML : 태그로 감싸는 방식

**XML(eXtensible Markup Language)**&#xC740; JSON보다 먼저 널리 쓰였던 데이터 포맷입니다.\
데이터를 **태그(꺾쇠 괄호)**&#xB85C; 감싸서 표현합니다.

같은 카페 주문을 XML로 표현하면 이렇습니다.

```xml
<주문>
  <메뉴>아이스 라떼</메뉴>
  <수량>2</수량>
  <얼음많이>true</얼음많이>
</주문>
```

| JSON               | XML                 |
| ------------------ | ------------------- |
| `{ }`, `[ ]` 기호 사용 | `<태그>` `</태그>` 사용   |
| 간결하고 가벼움           | 구조가 명확하지만 길어짐       |
| 최근 API의 주류         | 공공기관, 금융 등에서 여전히 사용 |

{% hint style="info" %}
최근 만들어지는 대부분의 서비스에서는 JSON을 사용합니다. 하지만 공공 데이터 포털이나 금융 시스템 등에서는 여전히 XML을 만날 수 있으므로, 형태를 알아두면 좋습니다.
{% endhint %}

***

### CSV : 표 형태의 데이터

**CSV(Comma-Separated Values)**&#xB294; 데이터를 **쉼표로 구분**하여 표 형태로 나타낸 포맷입니다.\
엑셀 파일을 텍스트로 열었을 때 보이는 형태라고 생각하면 됩니다.

```
이름,나이,회원여부
홍길동,25,true
김영희,30,false
이철수,28,true
```

| 특징 | 설명                                    |
| -- | ------------------------------------- |
| 구조 | 첫 줄이 제목(헤더), 나머지 줄이 데이터               |
| 장점 | 엑셀, 구글 시트 등에서 바로 열 수 있음               |
| 용도 | 대량의 데이터를 한꺼번에 내보내거나 가져올 때 (내보내기/가져오기) |

{% hint style="info" %}
CSV는 사람이 눈으로 읽기에도 편하고, 스프레드시트 프로그램에서 바로 열 수 있어서 데이터 이관이나 보고서 작업에 자주 사용됩니다.
{% endhint %}

***

### 한 줄 정리

> 데이터에는 텍스트(string), 숫자(number), 참/거짓(boolean), 배열(array), 객체(object) 같은 **타입**이 있고, JSON·XML·CSV 같은 **포맷**으로 포장되어 오갑니다. 현재 가장 널리 쓰이는 포맷은 JSON입니다.

이제 데이터가 어떻게 생겼는지 알았습니다.\
이 데이터를 받아서 실제로 처리하는 곳, **서버**의 세계로 들어가 봅시다.
