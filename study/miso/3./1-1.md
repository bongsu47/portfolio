# \[레벨 1] 파일을 첨부하여 이미지 생성하기

시중에서 제공되는 다양한 이미지 생성 서비스에서는 LLM에게 이미지를 생성하도록 요청할 때, 참고 이미지를 함께 첨부하여 비슷한 스타일이나 구도를 가진 이미지를 만들어 달라고 요청하는 방식이 많이 사용됩니다.&#x20;

이러한 방식은 원하는 결과에 더 가깝게 이미지를 생성할 수 있다는 장점이 있습니다. 이번 예제에서는 이미지를 첨부하여 유사한 스타일의 이미지를 생성하는 방법을 알아보겠습니다.

***

### 파일 타입 입력 변수 알아보기

1. **단일 파일 / 다중 파일**\
   파일을 첨부할 수 있는 변수는 **단일 파일**과 **다중 파일** 두 가지가 있습니다.\
   단일 파일은 한 개의 파일만 업로드할 수 있고, 다중 파일은 여러 개의 파일을 업로드할 수 있습니다.
2. **변수 이름과 입력 필드 이름의 차이**

* **변수 이름**: 영어만 사용 가능하며, 앱 내부에서 실제로 인식되는 변수 이름입니다.
* **입력 필드 이름**: 사용자가 앱을 사용할 때 화면에 표시되는 이름입니다. 예를 들어 _“파일을 입력해주세요”_ 와 같이 자유롭게 작성할 수 있습니다.

3. **입력 방식**

* **로컬 파일**: 내 PC에 있는 파일을 업로드하는 방식입니다.
* **URL**: 인터넷에 올라가 있는 파일의 URL을 입력하여 사용하는 방식입니다.

4. **파일 유형**\
   기본적으로 모든 파일 유형이 허용되어 있습니다.\
   특정 유형(예: **이미지**)만 활성화하면, 다른 유형의 파일을 업로드할 경우 오류가 발생합니다.
{% endhint %}

⚠️ 이전 예제에서 사용한 `input` 변수는 사용자가 항상 값을 입력하므로 필수 설정이 적절하지만, **파일은 사용자가 첨부하지 않는 경우도 있을 수 있습니다.**

파일 변수를 필수로 설정한 상태에서 파일을 업로드하지 않으면 **오류가 발생합니다.**

## 코드 노드

갑자기 코드 노드가 등장해서 당황하셨을 수 있습니다. 이 부분은 실제 기능 동작에 필수적인 과정은 아닙니다.

이전에 확인한 `json` 출력값에는 우리가 실제로 필요한 **파일 URL** 외에도 다양한 정보가 함께 포함되어 있어, 그대로 사용하면 형식이 다소 복잡하게 느껴질 수 있습니다. 따라서 이번 단계에서는 **필요한 URL만 추출해서 출력하도록** 구성해, 이후 단계에서 더 쉽게 활용할 수 있도록 했습니다.

즉, 이 작업은 기능을 새롭게 추가하거나 동작 방식에 영향을 주는 것이 아니라, **출력 결과를 더 단순하고 보기 좋게 정리해 사용성을 높이기 위한 과정**이라고 이해하시면 됩니다.
