{% hint style="info" %}
#### 텍스트와 문단이 다른건가요?

텍스트와 문단은 모두 **글자를 입력하는 형식**으로, 입력 방식이나 활용 방법에는 차이가 없습니다. **시작 노드에서 사용자 입력값**으로 받아 **다른 노드에 동일하게 전달**하여 사용할 수 있습니다. 차이점은 **입력 가능한 글자 수**에 있으며, 텍스트는 **최대 100자**까지, 문단은 **최대 10,000자**까지 입력할 수 있습니다.
{% endhint %}

## 나노바나나 도구 (Gemini flash 이미지)

미소에서는 **OpenAI 이미지 생성 도구**, **Google Imagen** 등 다양한 이미지 생성 도구를 제공합니다. 그중에서도 요즘 가장 핫하게 떠오르며 _가성비 끝판&#xC655;_&#xC73C;로 불리는 모델이 바로 **나노바나나**입니다. 🍌

**나노바나나 도구**는 나노바나나 모델을 기반으로 이미지를 생성하는 전용 이미지 생성 도구입니다. 간단한 설명만 입력해도 빠르고 퀄리티 높은 이미지를 만들어낼 수 있어, 실습용은 물론 실제 업무 활용에도 매우 유용합니다.

### 출력 형식 이해하기

나노바나나 도구는 이미지를 생성한 뒤, `text`에는 생성 결과에 대한 간단한 응답 메시지를 담아 전달하고, `json`에는 생성된 이미지의 URL을 담아 전달합니다.

<table><thead><tr><th width="116.63885498046875">출력값</th><th>설명</th></tr></thead><tbody><tr><td>text</td><td>이미지 생성에 대한 안내 텍스트입니다.</td></tr><tr><td>file</td><td>이 도구에서 파일 변수는 사용되지 않습니다.</td></tr><tr><td>json</td><td>생성된 이미지 파일의 정보 및 다운로드 링크가 포함되어 있습니다.</td></tr></tbody></table>

여기서 우리가 실제로 필요한 건 `file`이라는 변수입니다. 이 `file` 변수가 어떤 값을 담고 있고, 어떤 형태로 전달되는지 **조금 더 자세하게 들여다보겠습니다.**

```json
{
  "text": "**Analyzing the Composition**\n\nI'm now assessing the accurate representation of the scene, particularly the mountain details. I'm confirming that the geographical elements align with the user's provided description, including the viewpoint, key landmarks, and overall realism.\n\n\n\nimage has been created and sent to user already, you do not need to create it, just tell the user to check it now.",
  "files": [],
  "json": [

          "transfer_method": "tool_file",
          "remote_url": null,
          "related_id": "8718b2d7-c584-4c12-8a18-6aa5de6f2876",
          "upload_file_id": null,
          "filename": "840af1d9503e45f6bb650e9e7bbba6cb.png",

        }
      ]

```
{% endcode %}

겉으로 보면 복잡한 개발 언어처럼 보이지만, 위 구조는 우리가 앞으로 자주 마주치게 될 **JSON이라는 데이터 형식**입니다.

JSON은 한마디로, 정보를 정리해서 주고받기 위한 약속된 형식입니다.\
엑셀에서 “항목 – 값” 형태로 정리하듯이, JSON도 **이름: 값** 구조로 정보를 차곡차곡 담아두는 방식입니다.

예를 들어,

* `"type": "image"` → 이 파일의 종류는 이미지다
* `"filename": "example.png"` → 파일 이름은 example.png다
* `"url": "..."` → 실제로 다운로드할 수 있는 주소는 이것이다

이렇게 각각의 정보에 설명표를 붙여 정리해 둔 형태라고 이해하시면 됩니다.

기호가 많아 보여서 어렵게 느껴질 뿐, 본질은 단순합니다.\
“이건 무엇이고, 그 값은 무엇이다”라고 정리해 둔 구조일 뿐입니다.

비개발자라면 코드를 깊이 이해할 필요는 없습니다.\
다만, “아, 이건 정보를 구조화해서 정리한 형식이구나” 정도만 이해해도 AI 도구를 활용하거나 자동화를 설계할 때 훨씬 수월해집니다.

## 테스트하기

상단 중앙의 테스트하기 버튼을 누르면, 시작 단계의 input에 원하는 요청 사항을 입력할 수 있습니다. 입력을 완료한 뒤 워크플로우를 실행하면 이미지가 생성되며, 실행 결과에서 생성된 이미지의 다운로드 링크를 확인할 수 있습니다.

<figure><img src="../../../.gitbook/assets/image (50).png" alt="" width="563"><figcaption></figcaption></figure>

{% hint style="info" %}
#### 이미지가 url이 아닌 미리보기로 나와요

테스트 화면에서는 마크다운 형식을 자동으로 해석해 이미지를 **미리보기 형태로 보여주기 때문에 URL이 직접 표시되지 않습니다.**\
이미지가 보이면 마우스 오른쪽 버튼을 클릭해 “다른 이름으로 저장”을 선택하여 다운로드할 수 있습니다.

실제 앱에서 사용하는 **미리보기 환경에서는 이미지가 다운로드 링크(URL) 형태로 제공됩니다.**\
따라서 테스트 화면과 실제 사용 환경에서 보이는 방식이 조금 다를 수 있습니다.
{% endhint %}

{% hint style="info" %}
#### 생성했던 다운로드 링크가 다시 안들어가져요

미소에서 제공되는 모든 다운로드 링크는 **보안 정책에 따라 기본적으로 생성 후 약 10분이 지나면 자동으로 만료됩니다.**\
따라서 이미지를 다운로드하려면 생성된 직후 바로 저장하는 것이 좋습니다.
{% endhint %}
