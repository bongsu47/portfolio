# 워크플로우 앱 생성하기

## 앱 생성하기

1. 상단 메뉴에서 **플레이그라운드**를 클릭한 뒤 **앱 만들기**를 선택합니다.
2. AI 유형 선택 화면에서 **워크플로우**를 클릭합니다.
3. **이름**과 **설명**을 입력합니다.
4. **채팅 옵션을 비활성화(체크 해제)** 합니다.
5. **다음** 버튼을 클릭해 워크플로우 앱 생성을 완료합니다.

## 채팅 설정

워크플로우는 **일반 워크플로우**와 **대화형 워크플로우** 두 가지로 나뉘며,&#x20;

두 방식의 차이가 궁금하다면 아래에 안내된 워크플로우 매뉴얼을 차례대로 읽어보시면 쉽게 이해하실 수 있습니다.

[workflow](../../../manual/workflow/)
{% endcontent-ref %}

많은 분들이 두 가지 방식의 **차이**는 이해하고 계시지만, 실제로 **언제 어떤 워크플로우를 써야 하는지**에서 헷갈려하곤 합니다.

회의록 작성 예제를 기준으로 보면, 이 작업은 **대화형 워크플로우보다는 일반 워크플로우가 더 적합**합니다.

회의록 작성은 보통

* 정해진 입력값(회의 내용, 참석자, 날짜 등)을 한 번에 전달하고
* 미리 정의된 규칙에 따라
* 항상 비슷한 형식의 결과물을 생성하는 작업이기 때문입니다.

즉, 사용자가 AI와 주고받으며 추가 질문을 하거나 방향을 바꿀 필요 없이, **입력 → 처리 → 결과**의 흐름이 명확합니다.\
이런 경우에는 채팅 인터페이스 없이 자동으로 실행되는 일반 워크플로우가 더 단순하고 효율적입니다.

반대로 대화형 워크플로우는 사용자의 답변에 따라 다음 질문이 달라지거나, 여러 번의 상호작용을 통해 결과를 만들어야 할 때 적합합니다.

예를 들어 그동안 축적된 회의록 데이터를 기반으로 “지난 분기 주요 의사결정은 뭐였어?”, “A 안건은 왜 보류됐지?”처럼 사용자가 추가 질문을 이어가며 맥락을 좁혀가는 AI라면, 이러한 질의응답 흐름이 자연스럽게 이어지는 대화형 워크플로우가 더 적합했을 것입니다.

따라서 반복적으로 동일한 형식의 회의록을 빠르게 생성하는 목적이라면, 일반 워크플로우를 사용하는 것이 가장 알맞습니다.[^1]
