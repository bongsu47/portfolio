# 1. 반복되는 회의록 작성 자동화하기

### 한매니저의 고민

한매니저의 하루는 회의로 시작해서 회의록으로 끝납니다.

<figure><img src="../../../.gitbook/assets/image (70).png" alt="" width="375"><figcaption></figcaption></figure>

_"회의는 끝났는데, 정리는 늘 이제부터 시작이죠."_

회의가 끝나면 이런 요청이 바로 이어집니다.\
요약 공유, 결정사항 정리, 액션 아이템 도출, 담당자와 기한 확인, 그리고 메일 발송까지.\
미소 씨는 회의 내용을 다시 훑으며 핵심을 추리고 문서를 정리합니다.

문제는 이 과정이 매번 반복된다는 점입니다.\
회의가 많아질수록 정리 시간은 늘어나고, 공유가 늦어지면 실행도 함께 늦어집니다.\
결국 회의록 작성은 ‘정리 업무’가 아니라 팀의 속도를 좌우하는 병목이 됩니다.

이번 장에서는 **워크플로우의 기본 개념을 살펴보고,**&#x20;

**회의록을 업로드하면 정리된 문서를 회의 참가자들에게 이메일로 전송**하는 워크플로우를 구성해봅니다.
