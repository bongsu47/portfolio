# \[레벨 3] Google Sheet에 데이터 추가하기

\[레벨 3] 대량의 데이터 처리하기 예제에 이어서, 처리된 데이터를 Google Sheet에 추가해보겠습니다.

***

`values.append`는 **Google Sheet의 마지막 행 아래에 새로운 데이터를 추가하는 기능**입니다.\
기존 데이터를 수정하지 않고 **결과를 계속 쌓아 기록할 때** 사용합니다.

예를 들어 VOC 분석 결과를 Google Sheet에 저장하면, 새로운 결과가 나올 때마다 시트의 **맨 아래에 한 줄씩 자동으로 추가**됩니다.

즉, `values.append`는 **기존 데이터를 덮어쓰지 않고 새로운 데이터를 계속 추가하는 방식**입니다.

## 생각해보기

### append에서 사용하는 이중 배열 형식 이해하기

Google Sheet의 `values.append`는 데이터를 **이중 배열(2차원 배열)** 형식으로 전달해야 합니다.\
즉, 한 줄의 데이터라도 **행 배열을 다시 한 번 바깥 배열로 감싸는 구조**로 전달해야 합니다.

예를 들어 한 행의 데이터를 추가할 때는 다음과 같이 전달합니다.

```
[
  ["2026-03-09", "배송이 늦어요", "부정", "배송"]
]
```

<figure><img src="../../../.gitbook/assets/image (554).png" alt="" width="335"><figcaption></figcaption></figure>

여기서

* 안쪽 배열 `["2026-03-09", "배송이 늦어요", "부정", "배송"]` 은 **한 행(row)** 을 의미하고
* 바깥 배열은 **여러 행을 한 번에 추가할 수 있도록 감싸는 역할**을 합니다.

여러 행을 동시에 추가하려면 다음과 같은 형식이 됩니다.

```
[
  ["2026-03-09", "배송이 늦어요", "부정", "배송"],
  ["2026-03-09", "상담이 친절했어요", "긍정", "고객응대"]
]
```

<figure><img src="../../../.gitbook/assets/image (555).png" alt="" width="337"><figcaption></figcaption></figure>

만약 **한 열에만 데이터를 추가**하려면 각 행을 **값 하나만 가진 배열**로 만들어 전달하면 됩니다.\
이 경우에도 이중 배열 구조는 그대로 유지됩니다.

예를 들어 VOC 텍스트만 한 열에 계속 추가하려면 다음과 같이 전달합니다.

```
[
  ["배송이 늦어요"],
  ["상담이 친절했어요"],
  ["환불 처리가 오래 걸립니다"]
]
```

<figure><img src="../../../.gitbook/assets/image (556).png" alt="" width="143"><figcaption></figcaption></figure>

이렇게 하면 Google Sheet의 **한 열 아래로 데이터가 한 줄씩 추가**됩니다.

따라서 Google Sheet에 데이터를 추가하려면 **LLM의 출력 결과도 이 이중 배열 형식에 맞게 준비되어야 합니다.**

***

## 변수 추출기 노드

이전 예제에서 LLM 노드의 출력을 다시 살펴보겠습니다. 예제 데이터는 다음과 같은 구조로 이루어져 있습니다.

```
row_id,만족도,만족요인,페인포인트,고객니즈,핵심키워드,요약
1,만족,이벤트팬서비스|응원분위기,안내티켓|퇴장동선,앱 좌석안내 개선과 퇴장 동선 정비,이벤트|완벽한하루|퇴장동선,이벤트와 경기 모두 만족하나 안내·퇴장 동선 개선 필요
```

이 예제에서는 LLM을 통해 **6개의 인사이트를 추출**했습니다. 하지만 각각의 값을 별도의 변수로 생성한 것이 아니라, **`text`라는 하나의 변수에 문자열(String) 형태로 모두 담아 출력**했습니다.

LLM 노드는 프롬프트에 따라 다양한 형식의 결과를 생성할 수 있지만, 기본적으로 **모든 결과가 `text`라는 단일 변수로 묶여 출력되는 특징**이 있습니다.

따라서 결과 안에 여러 정보가 포함되어 있더라도, 실제 워크플로우에서 활용하려면 **각 항목을 다시 분리하는 추가 작업**이 필요합니다.일반적으로는 코드 노드를 사용해 문자열을 파싱하거나 구조를 나누는 과정을 거쳐야 합니다.

이러한 번거로움을 줄이기 위해 제공되는 기능이 **변수 추출기(Parameter Extractor)** 입니다.

변수 추출기를 사용하면 **미리 변수의 이름과 타입을 정의**할 수 있습니다.\
그러면 LLM이 결과를 생성할 때 해당 구조에 맞게 **여러 개의 변수로 나누어 결과를 출력**합니다.



**예를 들어 이메일을 생성한다고 가정해 보겠습니다.**

<figure><img src="../../../.gitbook/assets/image (593).png" alt="" width="375"><figcaption></figcaption></figure>

일반적으로 LLM 노드만 사용할 경우 결과는 다음과 같이 **하나의 text 안에 함께 출력**됩니다.

메일 제목: 회의 일정 안내\
메일 본문: 안녕하세요. 내일 오후 2시에 팀 회의가 예정되어 있습니다. 회의실 A에서 진행되며 관련 자료를 미리 확인해 주세요.

이 경우 제목과 본문이 하나의 문자열에 포함되어 있기 때문에,\
워크플로우에서 **메일 제목만 따로 사용하거나 메일 본문만 따로 활용하기가 어렵습니다.**

하지만 **변수 추출기**를 사용하면 결과가 다음과 같이 **각각의 변수로 분리되어 출력**됩니다.

* 메일제목
* 메일본문

즉, 메일 제목은 **메일제목 변수**로, 메일 본문은 **메일본문 변수**로 각각 생성됩니다.

이처럼 결과가 구조화되어 있으면 이후 워크플로우에서

* 메일 제목은 **제목 입력 필드**에
* 메일 본문은 **내용 입력 필드**에

각각 필요한 값만 선택해 사용할 수 있습니다.

또한 변수 추출기는 `string`뿐만 아니라 array(string) 등 미소에서 지원하는 다양한 타입으로도 결과를 생성할 수 있습니다. 덕분에 별도의 코드 처리 없이도 **구조화된 데이터를 바로 활용할 수 있다는 장점**이 있습니다.

### STEP 1. 변수 추출 노드 구성하기

1. 이전 예제의 상태에서 반복 노드 내부에 배치한 LLM 노드에 마우스 우클릭 후 노드 삭제를 클릭합니다.

<figure><img src="../../../.gitbook/assets/image (591).png" alt="" width="375"><figcaption></figcaption></figure>

2. 반복 노드 내부의 \[+] 버튼을 클릭하여 변수 추출 노드를 배치합니다.
3. 변수 추출 노드는 아래와 같이 구성되어 있습니다.
   1. 입력 변수 : LLM의 입력으로 사용할 변수를 할당합니다. 할당된 변수는 자동으로 LLM에게 데이터가 전달됩니다.
   2. 비전 : 이미지를 첨부할 경우 활성화하여 파일 변수를 할당합니다.
   3. \+ 버튼 : 출력 변수를 각각 정의합니다.
   4. 지시 : 출력 변수에 대한 각 지시 프롬프트를 입력합니다. 출력 변수마다 구조화하는게 좋습니다.

<figure><img src="../../../.gitbook/assets/image (592).png" alt="" width="563"><figcaption></figcaption></figure>

4. 먼저 입력 변수를 선택합니다. 반복문의 item 변수로 할당합니다.
5. 매개 변수 추출 항목 오른쪽 끝의 \[+] 버튼을 클릭하면 출력 변수를 설정할 수 있는 UI가 표시됩니다. 여기에서 변수의 **이름, 타입, 설명**을 입력하면 됩니다. 이 정보는 기본적으로 LLM이 함께 인식하므로, 각 항목을 가능한 한 구체적이고 상세하게 작성하는 것이 좋습니다.

<figure><img src="../../../.gitbook/assets/image (595).png" alt="" width="563"><figcaption></figcaption></figure>

4. 이번 예제에서는 편의를 위해서 만족도와 요약만 추가해보겠습니다.
   1. 만족도
      1. 매개 변수 이름 : satisfaction
      2. 타입 : 문자열
      3. 설명 : 관람객 경험의 전반적 만족도 분류 값 이다. 매우만족|만족|보통|불만|매우불만 중 하나를 출력한다.
   2. 요약
      1. 매개 변수 이름 : summary
      2. 타입 : 문자열
      3. 설명 : VOC 내용을 40자 이내 한 줄로 요약한 문장이다.
5. 지시 프롬프트를 복사해 입력합니다. 아래 예시를 그대로 사용하기보다는 미소AI를 활용해 자신의 목적에 맞게 수정하여 사용하는 것을 권장합니다.

{% code expandable="true" %}
```
<Role>
스포츠 경기장 관람객 VOC를 감정 톤과 맥락 기반으로 분석하는 전문 애널리스트
</Role>

<Task>
관람객 설문 데이터에서 다음 두 변수를 추출:
1. satisfaction: 관람 경험 만족도 (5단계)
2. summary: VOC 핵심 요약 (40자 이내)
</Task>

<Input>
의견, 상세불만족, 개선방법, 가장불만족, 추천점수, 재방문의사 등 모든 필드를 종합 분석
※ 텍스트 감정이 추천점수와 충돌 시 텍스트 우선
</Input>

<Output>
<satisfaction>
- 매우만족: 강한 감탄/추천 표현 ("최고", "완벽", "꼭 가보세요")
- 만족: 긍정 표현 ("좋았다", "만족", "즐거웠다")
- 보통: 중립적 서술 또는 긍정·부정 혼재
- 불만: 불편/아쉬움 표현 ("불편했다", "개선 필요")
- 매우불만: 강한 부정 표현 ("최악", "다시 안 온다")
</satisfaction>

<summary>
- 40자 이내
- 핵심 만족/불만 요소 포함 (시설, 서비스, 좌석 등)
- 간결하고 자연스러운 문장

예시:
"좌석 시야 좋고 직원 친절해 만족스러운 관람"
"화장실 부족과 긴 대기시간으로 불편 경험"
</summary>
</Output>

<Rules>
- satisfaction은 5개 값 중 하나만 사용
- 텍스트 데이터만 기반으로 판단 (추측 금지)
- 모호한 경우 '보통'으로 분류
- 개인정보 제외
</Rules>

<Output_Format>
- "satisfaction": "매우만족|만족|보통|불만|매우불만"
- "summary": "40자 이내 요약"
</Output_Format>
```
{% endcode %}

## Google Sheet 도구 노드

### STEP 2. values.append 도구 노드 추가하기

1. 반복 블록 내 변수 추출 노드의 우측 \[+] 버튼을 클릭하여 `values.append` 노드를 추가합니다.

<figure><img src="../../../.gitbook/assets/image (539).png" alt="" width="375"><figcaption></figcaption></figure>

2. 새롭게 추가될 시트의 범위를 설정해야 합니다. 현재는 `data` 시트만 있으므로 Google Sheet로 이동해 새로운 시트를 추가합니다. 여기서는 `analysis`라는 이름의 시트를 추가하겠습니다.

<figure><img src="../../../.gitbook/assets/image (588).png" alt="" width="239"><figcaption></figcaption></figure>

3. 시트의 헤더도 미리 입력합니다. '만족도', '요약' 으로 입력하겠습니다.

<figure><img src="../../../.gitbook/assets/image (540).png" alt="" width="246"><figcaption></figcaption></figure>

3. 다시 **Append 노드**로 이동합니다.&#x20;

* **Sheet ID**는 이전 단계에서 사용한 값과 동일하게 설정하고, **Range**는 새로 만든 **analysis 시트**를 사용하도록 설정합니다.&#x20;
* Range는 입력될 열의 개수를 기준으로 설정해야 합니다. 이번 실습에서는 변수 추출기를 통해 **satisfaction**과 **summary** 두 개의 변수를 생성했으므로, Google Sheet에는 **2개의 열**이 필요합니다.
* 따라서 Range는 **analysis!A:B**로 설정합니다.



4. **values** 에 변수 추출기에서 생성된 각 변수를 이중 배열 형태로 입력합니다.

{% code expandable="true" %}
```
[["변수1","변수2"]]
```
{% endcode %}

<figure><img src="../../../.gitbook/assets/Clipboard-20260309-153837-075.gif" alt="" width="563"><figcaption></figcaption></figure>

### STEP 3. 반복 노드 설정하기

사실 미소에서 직접 데이터 출력을 확인하는 것이 아니라 Google Sheet에 데이터를 입력하는 방식이기 때문에, 어떤 출력 변수를 선택하더라도 큰 차이는 없습니다. 다만 출력 변수 할당은 필수이기 때문에, Google Sheet 도구의 출력 변수를 선택하여 할당했습니다.

<figure><img src="../../../.gitbook/assets/image (542).png" alt="" width="375"><figcaption></figcaption></figure>

### STEP 4. 종료 노드 설정하기

종료 노드에서도 변수 할당은 필수가 아닌 선택 사항입니다. 별도의 변수 할당 없이 단순히 연결만 해주셔도 됩니다.
